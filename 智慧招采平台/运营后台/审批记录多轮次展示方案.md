# 审批记录多轮次展示方案

---

## 背景说明

当前审批记录页面仅包含单次审批记录。实际业务中，一次信息审核（如单位信息变更）可能经历"不通过 → 重新编辑 → 再次提交"的循环，产生多条审批记录。每条记录又包含：发起人、单位、单位内部多个审批节点，信息复杂度高。

本文档定义审批记录多轮次展示的产品方案。

---

## 业务场景约束

| 维度 | 实际情况 |
| :--- | :--- |
| 重审轮次 | ≤3轮（通常1-2轮即可通过） |
| 对比需求 | 需要对比不同轮次，但不频繁 |
| 展示形式 | 弹窗（空间有限） |

---

## 方案：时间轴折叠式

### 核心设计理念

以 **审批轮次（Round）** 为一级分组维度，采用可折叠面板组织多轮审批记录。每轮内部沿用现有的 **按单位分组** 视觉方案，形成"轮次 → 单位 → 节点"三层信息结构。

设计关键词：时间倒序、最新展开、历史收起、渐进式披露

### 整体视觉布局

```
┌─ 📜 审批记录 ────────────────────────────────────────────────┐
│                                                              │
│  ┌─ ▼ 第3轮审批 | 2026-04-20 09:30提交 | 🔵 审批中 ───────┐ │
│  │                                                        │ │
│  │  发起人：张三 | 修改说明：已重新整理所有材料              │ │
│  │  ────────────────────────────────────────────────      │ │
│  │                                                        │ │
│  │  ┌─ [🔵 XX三级公司 (审批中)] ──────────────────┐      │ │
│  │  │ ▶ 04-20 09:35 | 部门审批 | 王五(处理中)      │      │ │
│  │  │ ▶ --         | 负责人审批 | --(待指定)       │      │ │
│  │  └────────────────────────────────────────────┘      │ │
│  │  ┌─ [⚪ XX二级公司 (未到达)] ──────────────────┐      │ │
│  │  │ ▶ 节点1：部门审批                           │      │ │
│  │  │ ▶ 节点2：总经理审批                         │      │ │
│  │  └────────────────────────────────────────────┘      │ │
│  └──────────────────────────────────────────────────────┘ │
│                                                              │
│  ┌─ ▶ 第2轮审批 | 2026-04-18 14:00提交 | ❌ 不通过 ───────┐ │
│  │  驳回原因：营业执照复印件未加盖公章...                   │ │
│  └──────────────────────────────────────────────────────┘ │
│                                                              │
│  ┌─ ▶ 第1轮审批 | 2026-04-15 10:00提交 | ❌ 不通过 ───────┐ │
│  │  驳回原因：缺少法人授权委托书                            │ │
│  └──────────────────────────────────────────────────────┘ │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 轮次标题栏字段

| 字段 | 展示规则 | 示例 |
| :--- | :--- | :--- |
| **轮次编号** | 固定格式 `第N轮审批` | 第1轮审批、第2轮审批 |
| **提交时间** | 发起人提交该轮审批的时间 | 2026-04-20 09:30 |
| **轮次状态** | 该轮审批的最终结果 | ✅ 已通过 / ❌ 不通过 / 🔵 审批中 / ⚫ 已撤销 |
| **驳回原因摘要** | 仅在不通过时显示，截取前30字 | 驳回原因：营业执照复印件未加盖公章... |
| **撤销信息摘要** | 仅在已撤销时显示 | 撤销人：张三 | 撤销时间：04-16 11:00 |

### 轮次默认展开/收起规则

| 轮次状态 | 默认行为 | 说明 |
| :--- | :--- | :--- |
| **审批中（最新一轮）** | **默认展开** | 当前正在进行的审批，用户最关注 |
| **已通过** | 默认收起 | 历史记录，按需查看 |
| **不通过** | 默认收起 | 历史记录，摘要已显示驳回原因 |
| **已撤销** | 默认收起 | 历史记录，摘要已显示撤销信息 |

### 展开/收起交互

- 点击轮次标题栏任意区域触发展开/收起
- 展开箭头图标：`▼` 展开 / `▶` 收起
- 同一时间允许多个轮次同时展开（非手风琴模式），方便对比

### 轮次内部展示

展开后顶部显示概要信息：

| 字段 | 说明 | 示例 |
| :--- | :--- | :--- |
| **发起人** | 该轮审批的发起人 | 张三 |
| **提交时间** | 该轮审批的提交时间 | 2026-04-18 14:00 |
| **修改说明** | 可选字段，发起人重新提交时填写的修改说明 | 已补充营业执照复印件并加盖公章 |

下方沿用现有按单位分组卡片设计：

- ✅ 已完成单位：绿色实心对钩，默认收起
- 🔵 当前单位：蓝色呼吸/高亮，默认展开
- ⚪ 未到达单位：灰色虚线/占位，默认收起
- ❌ 不通过单位：红色高亮，默认展开（聚焦驳回节点）

### 截断逻辑

- **审批中**：展示全量节点（含未到达的灰色占位）
- **不通过**：截断至驳回节点，后续节点不渲染
- **已撤销**：仅展示"提交"和"撤销"两个节点，不展示单位分组卡片

### 已撤销轮次内部展示

已撤销轮次展开后，不展示单位分组卡片，仅展示一条简洁的撤销记录：

```
┌─ ▼ 第2轮审批 | 2026-04-15 10:00提交 | ⚫ 已撤销 ────────────┐
│                                                              │
│  发起人：张三 | 提交时间：2026-04-15 10:00                   │
│  ──────────────────────────────────────────────────────      │
│                                                              │
│  ● 2026-04-15 10:00  提交审批                                │
│  ● 2026-04-16 11:00  已撤销（撤销人：张三）                   │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

| 字段 | 说明 | 示例 |
| :--- | :--- | :--- |
| **提交时间** | 该轮审批的提交时间 | 2026-04-15 10:00 |
| **撤销人** | 执行撤销操作的用户 | 张三 |
| **撤销时间** | 撤销操作的时间 | 2026-04-16 11:00 |
| **撤销原因** | 不需要 | — |

### 边界场景

- **仅一轮审批**：不显示轮次折叠面板，直接展示单位分组卡片，避免无意义的额外层级
- **发起人变更**：不同轮次可能由不同发起人提交，在轮次头部信息区明确显示该轮的实际发起人

### 交互细节

- 非手风琴模式，允许多个轮次同时展开（方便对比）
- 驳回原因支持悬停显示完整内容（Tooltip）
- 点击驳回原因可直接展开该轮次并定位至驳回节点
- 页面加载时自动滚动至最新一轮（默认展开的区域）

### 弹窗适配要点

- 弹窗建议宽度 ≥ 720px，高度 ≥ 600px，内容区可滚动
- 轮次标题栏固定高度，确保折叠/展开交互流畅
- 弹窗内滚动时，最新一轮始终可见（sticky定位或自动滚动）

## 数据结构设计

### 数据层级

在现有 `sys_approval_instance_node` 基础上，新增轮次维度：

```
审批业务单据 (Business Record)
  └── 审批轮次 (Approval Round)
        ├── round_no          轮次编号 (1, 2, 3...)
        ├── submitter_uid     发起人
        ├── submit_time       提交时间
        ├── modify_note       修改说明（可选）
        ├── round_status      轮次状态 (RUNNING / APPROVED / REJECTED / CANCELLED)
        ├── reject_reason     驳回原因摘要
        ├── cancelled_by      撤销人UID（可选）
        ├── cancelled_time    撤销时间（可选）
        └── 审批实例 (Approval Instance)
              └── sys_approval_instance_node (现有表结构不变)
```
