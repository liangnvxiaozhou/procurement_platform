# 审批记录多轮次展示方案

---

## 背景说明

当前审批记录页面仅包含单次审批记录。实际业务中，一次信息审核（如单位信息变更）可能经历"不通过 → 重新编辑 → 再次提交"的循环，产生多条审批记录。每条记录又包含：发起人、单位、单位内部多个审批节点，信息复杂度高。

本文档提供两种展示方案供业务方评估选择。

---

## 业务场景约束

| 维度 | 实际情况 |
| :--- | :--- |
| 重审轮次 | ≤3轮（通常1-2轮即可通过） |
| 对比需求 | 需要对比不同轮次，但不频繁 |
| 展示形式 | 弹窗（空间有限） |

---

## 方案 A：时间轴折叠式

### 核心设计理念

以 **审批轮次（Round）** 为一级分组维度，采用可折叠面板组织多轮审批记录。每轮内部沿用现有的 **按单位分组** 视觉方案，形成"轮次 → 单位 → 节点"三层信息结构。

设计关键词：时间倒序、最新展开、历史收起、渐进式披露

### 整体视觉布局

```
┌─ 📜 审批记录 ────────────────────────────────────────────────┐
│                                                              │
│  ┌─ ▼ 第3轮审批 | 2026-04-20 09:30提交 | 🔵 审批中 ───────┐ │
│  │                                                        │ │
│  │  发起人：张三 | 修改说明：已重新整理所有材料              │ │
│  │  ────────────────────────────────────────────────      │ │
│  │                                                        │ │
│  │  ┌─ [🔵 XX三级公司 (审批中)] ──────────────────┐      │ │
│  │  │ ▶ 04-20 09:35 | 部门审批 | 王五(处理中)      │      │ │
│  │  │ ▶ --         | 负责人审批 | --(待指定)       │      │ │
│  │  └────────────────────────────────────────────┘      │ │
│  │  ┌─ [⚪ XX二级公司 (未到达)] ──────────────────┐      │ │
│  │  │ ▶ 节点1：部门审批                           │      │ │
│  │  │ ▶ 节点2：总经理审批                         │      │ │
│  │  └────────────────────────────────────────────┘      │ │
│  └──────────────────────────────────────────────────────┘ │
│                                                              │
│  ┌─ ▶ 第2轮审批 | 2026-04-18 14:00提交 | ❌ 不通过 ───────┐ │
│  │  驳回原因：营业执照复印件未加盖公章...                   │ │
│  └──────────────────────────────────────────────────────┘ │
│                                                              │
│  ┌─ ▶ 第1轮审批 | 2026-04-15 10:00提交 | ❌ 不通过 ───────┐ │
│  │  驳回原因：缺少法人授权委托书                            │ │
│  └──────────────────────────────────────────────────────┘ │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 轮次标题栏字段

| 字段 | 展示规则 | 示例 |
| :--- | :--- | :--- |
| **轮次编号** | 固定格式 `第N轮审批` | 第1轮审批、第2轮审批 |
| **提交时间** | 发起人提交该轮审批的时间 | 2026-04-20 09:30 |
| **轮次状态** | 该轮审批的最终结果 | ✅ 已通过 / ❌ 不通过 / 🔵 审批中 |
| **驳回原因摘要** | 仅在不通过时显示，截取前30字 | 驳回原因：营业执照复印件未加盖公章... |

### 轮次默认展开/收起规则

| 轮次状态 | 默认行为 | 说明 |
| :--- | :--- | :--- |
| **审批中（最新一轮）** | **默认展开** | 当前正在进行的审批，用户最关注 |
| **已通过** | 默认收起 | 历史记录，按需查看 |
| **不通过** | 默认收起 | 历史记录，摘要已显示驳回原因 |

### 展开/收起交互

- 点击轮次标题栏任意区域触发展开/收起
- 展开箭头图标：`▼` 展开 / `▶` 收起
- 同一时间允许多个轮次同时展开（非手风琴模式），方便对比

### 轮次内部展示

展开后顶部显示概要信息：

| 字段 | 说明 | 示例 |
| :--- | :--- | :--- |
| **发起人** | 该轮审批的发起人 | 张三 |
| **提交时间** | 该轮审批的提交时间 | 2026-04-18 14:00 |
| **修改说明** | 可选字段，发起人重新提交时填写的修改说明 | 已补充营业执照复印件并加盖公章 |

下方沿用现有按单位分组卡片设计：

- ✅ 已完成单位：绿色实心对钩，默认收起
- 🔵 当前单位：蓝色呼吸/高亮，默认展开
- ⚪ 未到达单位：灰色虚线/占位，默认收起
- ❌ 不通过单位：红色高亮，默认展开（聚焦驳回节点）

### 截断逻辑

- **审批中**：展示全量节点（含未到达的灰色占位）
- **不通过**：截断至驳回节点，后续节点不渲染

### 边界场景

- **仅一轮审批**：不显示轮次折叠面板，直接展示单位分组卡片，避免无意义的额外层级
- **发起人变更**：不同轮次可能由不同发起人提交，在轮次头部信息区明确显示该轮的实际发起人

### 交互细节

- 非手风琴模式，允许多个轮次同时展开（方便对比）
- 驳回原因支持悬停显示完整内容（Tooltip）
- 点击驳回原因可直接展开该轮次并定位至驳回节点
- 页面加载时自动滚动至最新一轮（默认展开的区域）

### 弹窗适配要点

- 弹窗建议宽度 ≥ 720px，高度 ≥ 600px，内容区可滚动
- 轮次标题栏固定高度，确保折叠/展开交互流畅
- 弹窗内滚动时，最新一轮始终可见（sticky定位或自动滚动）

## 方案 D：双层嵌套式

### 核心设计理念

将审批记录分为 **两层视图**：
- **外层（轮次摘要层）**：以卡片列表展示每轮审批的核心摘要信息
- **内层（审批详情层）**：点击轮次卡片后，在独立面板中展示该轮的完整审批流

设计关键词：摘要优先、按需深入、视图分离、焦点明确

### 整体视觉布局

```
┌─ 📜 审批记录 ────────────────────────────────────────────────────┐

│  ┌─ 📋 轮次列表 ────────────┐  ┌─ 🔍 第3轮审批详情 ──────────┐  │

│  │ ┌──────────────────────┐ │  │ ┌─ [🔵 XX三级公司] ──────┐  │  │
│  │ │ 第3轮审批  🔵 审批中 │ │  │ │ ▶ 04-20 09:35          │  │  │
│  │ │ 发起人：张三         │ │  │ │   部门审批 | 王五       │  │  │
```

### 外层：轮次摘要卡片

每张卡片包含以下信息：

| 字段 | 说明 | 示例 |
| :--- | :--- | :--- |
| **轮次编号** | 第N轮审批 | 第3轮审批 |
| **轮次状态** | 醒目的状态标签 | 🔵 审批中 / ❌ 不通过 / ✅ 已通过 |
| **发起人** | 该轮发起人 | 张三 |
| **提交时间** | 该轮提交时间 | 2026-04-20 09:30 |
| **修改说明** | 重新提交时填写 | 已重新整理所有材料 |
| **当前进度** | 审批中时显示 | XX三级公司 - 部门审批(处理中) |
| **驳回原因** | 不通过时显示 | 营业执照复印件未加盖公章 |
| **驳回节点** | 不通过时显示 | XX二级公司 - 部门审批(赵六) |

### 内层：审批详情面板

- 点击轮次卡片后，在右侧独立面板展示该轮完整审批流
- 完全复用现有按单位分组卡片设计
- 默认选中最新一轮，自动展示其详情

### 布局模式

鉴于审批记录以弹窗形式展示，采用 **左右分栏** 布局：

| 区域 | 宽度占比 | 说明 |
| :--- | :--- | :--- |
| **左侧：轮次列表** | 约 35% | 固定宽度，可滚动，展示所有轮次摘要卡片 |
| **右侧：审批详情** | 约 65% | 展示选中轮次的完整审批流，可滚动 |

### 边界场景

- **仅一轮审批**：不显示左侧轮次列表，直接展示审批详情（全屏宽度）
- **发起人变更**：摘要卡片显示该轮实际发起人
- **快速对比**：支持在新窗口中打开某轮详情，方便多轮对比

### 交互细节

- 轮次卡片点击即选中，高亮显示当前选中状态
- 默认选中最新一轮，自动加载详情
- 轮次列表支持按状态筛选（全部/审批中/不通过/已通过）
- 支持"仅看驳回轮次"快速筛选
- 左侧列表和右侧详情各自独立滚动

### 弹窗适配要点

- 弹窗建议宽度 ≥ 960px，高度 ≥ 600px
- 左侧轮次列表固定宽度（约 320px），右侧详情自适应
- 如果弹窗宽度受限，可改为上下布局（上方轮次列表横向滚动，下方详情）

## 方案对比

| 维度 | 方案A：时间轴折叠式 | 方案D：双层嵌套式 |
| :--- | :--- | :--- |
| **信息层级** | 三层嵌套（轮次→单位→节点） | 两层视图分离（摘要→详情） |
| **首屏信息量** | 中等（标题栏+展开的一轮详情） | 较大（所有轮次摘要可见） |
| **页面长度** | 展开后较长，需滚动 | 固定高度，各区域独立滚动 |
| **多轮对比** | 方便（可同时展开多轮） | 不便（一次只看一轮详情） |
| **视觉复杂度** | 较高（嵌套折叠） | 较低（视图分离，焦点明确） |
| **交互成本** | 低（点击展开即可） | 中等（需切换选中轮次） |
| **弹窗最小宽度** | ≥ 720px | ≥ 960px（左右分栏） |
| **实现复杂度** | 中等 | 中等偏高（需维护双面板联动） |
| **移动端适配** | 较难（嵌套结构复杂） | 较易（可改为上下布局） |

---

## 推荐建议

基于当前业务场景（≤3轮、需要对比但不频繁、弹窗展示）：

**推荐方案 A（时间轴折叠式）**，理由如下：

1. **轮次少（≤3轮）**：折叠面板展开后内容量可控，不会造成页面过长
2. **对比需求**：非手风琴模式支持同时展开多轮，满足偶尔对比需求
3. **弹窗友好**：所需弹窗宽度更小（720px vs 960px），在常见屏幕分辨率下更友好
4. **实现成本低**：基于现有按单位分组方案，仅需增加轮次折叠层，改动量小
5. **交互直观**：展开/收起是用户熟悉的交互模式，学习成本低

方案 D 更适合轮次较多（>5轮）、需要快速浏览摘要筛选的场景，当前业务场景下其优势不明显，而弹窗宽度要求更高。

## 数据结构设计

### 数据层级

在现有 `sys_approval_instance_node` 基础上，新增轮次维度：

```
审批业务单据 (Business Record)
  └── 审批轮次 (Approval Round)
        ├── round_no          轮次编号 (1, 2, 3...)
        ├── submitter_uid     发起人
        ├── submit_time       提交时间
        ├── modify_note       修改说明（可选）
        ├── round_status      轮次状态 (RUNNING / APPROVED / REJECTED)
        ├── reject_reason     驳回原因摘要
        └── 审批实例 (Approval Instance)
              └── sys_approval_instance_node (现有表结构不变)
```
