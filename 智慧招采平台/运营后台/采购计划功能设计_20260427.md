# 采购计划功能设计

> 设计日期：2026-04-27
> 状态：需求梳理完成

---

## 一、功能概述

采购计划模块用于对集团的采购计划进行制定和管理，为企业提供及时准确的采购计划和执行路线。

本模块支持企业建立年度采购计划、变更子计划及采购项目立项；支持二级单位审核控股投资企业年度采购计划及分派集中采购计划。

### 模块划分

```
采购计划管理
│
├── 1. 本企业年度计划（编制端）
│   ├── 年度计划类型：年度计划 → 子计划（自行/集采）→ 明细
│   └── 项目计划类型：项目计划（仅自行采购）→ 明细
│
├── 2. 所属企业年度计划（查看+分派端）
│   └── 仅展示下级单位已生效的计划（年度计划/项目计划）
│   └── 支持分派集采子计划给执行单位
│
├── 3. 本企业可执行计划（执行端）
│   ├── 本企业审核通过的子计划/项目计划
│   ├── 二级单位分派的集采子计划
│   └── 支持变更明细，重新提交审核
│
└── 4. 采购项目立项（立项端）
    └── 基于可执行计划中的子计划/项目计划进行立项
    └── 拟采购方式自动带入，可编辑

审核入口：采购系统 → 任务中心（统一处理所有计划审核）
```

## 二、数据结构

### 2.1 年度计划结构

```
年度计划（主表 plan_annual）
  └── 子计划（plan_annual_item）
        ├── 计划类型：自行采购 / 集中采购
        ├── 项目类型：工程 / 物资 / 服务
        ├── 拟采购方式：公开招标 / 邀请招标 / 询比 / 谈判 / 直接 / 竞价
        ├── 项目明细（工程/服务类）
        └── 物资明细（物资类）
```

### 2.2 项目计划结构

```
项目计划（主表 plan_project）
  ├── 项目类型：工程 / 物资 / 服务
  ├── 拟采购方式：公开招标 / 邀请招标 / 询比 / 谈判 / 直接 / 竞价
  ├── 项目明细（工程/服务类）
  └── 物资明细（物资类）
```

**注**：项目计划无"计划类型"字段，默认自行采购，不参与集采分派。

## 三、核心业务流程

### 3.1 编制流程

```
年度计划：
  创建年度计划主表 → 添加子计划 → 添加明细 → 提交审核

项目计划：
  创建项目计划主表 → 添加明细 → 提交审核
```

### 3.2 审核流程（在任务中心处理）

```
年度计划：
  内部审核 → 逐级审核 → 二级单位审核 → 生效

项目计划/自行采购计划：
  内部审核 → 生效
```

### 3.3 分派流程（仅集采）

```
二级单位查看已生效集采子计划 → 选择执行单位 → 分派
```

### 3.4 执行流程

```
查看可执行计划 → 变更明细（需审核）→ 立项
```

### 3.5 立项流程

```
选择可执行计划中的子计划 → 创建采购项目

约束：
  - 一个子计划仅用一次
  - 多子计划项目类型须一致
  - 拟采购方式自动带入，可编辑
```

| 字段 | 必填 | 说明 |

### 4.2 所属企业年度计划

**功能路径**：`采购系统 → 计划管理 → 所属企业年度计划`

**功能定位**：二级单位查看下级单位已生效的计划，支持分派集采子计划。

**注**：审核操作不在本页面，统一在"采购系统 → 任务中心"中处理。

#### 列表页

```
┌─ 所属企业年度计划 ────────────────────────────────────────┐
│                                                           │
│ 筛选：[ 下属单位 ▼ ] [ 计划年度 ▼ ] [ 计划类型 ▼ ] [ 搜索 ]│
│                                                           │
│ 单位 | 计划名称 | 计划类型 | 预算金额 | 子计划数 | 状态  │
│ ────────────────────────────────────────────────────────
│ A公司| 2025年度计划 | 年度计划 | 5000万 | 12 | 已生效 │
│ B公司| 设备采购     | 项目计划 | 1200万 | -  | 已生效 │
│                                                           │
│ 操作：查看详情                                           │
└───────────────────────────────────────────────────────────┘
```

- **筛选条件**：下属单位、计划年度、计划类型（年度计划/项目计划）
- **列表字段**：单位名称、计划名称、计划类型、预算金额、子计划数、状态
- **数据范围**：仅展示下级单位已生效（审核通过）的计划
- **操作按钮**：查看详情

#### 详情页

- 展示计划完整信息（主表 + 子计划 + 明细）
- 子计划列表中标注"集中采购"的子计划，显示"分派"按钮

#### 分派操作（弹窗）

```
┌─ 分派集中采购计划 ────────────────────────────────────────┐
│                                                           │
│ 子计划：电脑采购（A公司，50万）                            │
│ 明细：电脑 100台                                          │
│                                                           │
│ * 执行单位：[ 选择下属单位 ▼ ]                            │
│   （仅限当前二级单位下属的三级单位）                       │
│                                                           │
│   备注：[ 由XX公司统一采购，量大优惠 ]                    │
│                                                           │
│ [ 确认分派 ]  [ 取消 ]                                    │
└───────────────────────────────────────────────────────────┘
```

**分派规则**：
- 仅可分派计划类型 = "集中采购" 且已生效的子计划
- 执行单位仅限当前二级单位下属的单位
- 分派后，子计划出现在执行单位的"本企业可执行计划"中
- 执行单位可基于此子计划进行立项

**功能路径**：`采购系统 → 计划管理 → 本企业可执行计划`

**功能定位**：查看已生效的子计划/项目计划，支持变更明细。

#### 列表页

```
┌─ 本企业可执行计划 ────────────────────────────────────────┐
│                                                           │
│ 来源：[ 全部 ▼ ]（本企业/集采分派）                       │
│ 筛选：[ 计划名称 ] [ 计划类型 ▼ ] [ 搜索 ]                │
│                                                           │
│ 计划名称 | 来源 | 估算金额 | 项目类型 | 已立项 | 操作     │
│ ────────────────────────────────────────────────────────
│ 办公楼改造 | 本企业   | 800万  | 工程 | 否 | 查看/变更/立项│
│ 电脑采购   | 集采分派 | 50万   | 物资 | 否 | 查看/变更/立项│
│ 设备采购   | 本企业   |1200万  | 工程 | 是 | 查看         │
│                                                           │
└───────────────────────────────────────────────────────────┘
```

- **来源筛选**：全部 / 本企业 / 集采分派
- **列表字段**：计划名称、来源、估算金额、项目类型、已立项状态、操作
- **操作按钮**：
  - 未立项：查看 / 变更 / 立项
  - 已立项：查看

#### 变更操作

**交互流程**：

```
可执行计划列表 → 点击"变更" → 新页面打开编辑表单 → 
修改明细 → 提交审核 → 进入任务中心待审核 → 
审核通过 → 可执行计划更新
```

**变更页面**：

- 新页面打开，展示子计划/项目计划信息（只读）
- 明细列表支持：新增 / 编辑 / 删除
- 修改后提交审核
- **关键规则**：
  - 本企业年度计划中的原始记录不变
  - 变更审核通过后，可执行计划中的明细更新
  - 变更期间，原明细继续有效

### 4.4 采购项目立项

**功能路径**：`采购系统 → 采购立项 → 项目立项`

**功能定位**：基于可执行计划创建采购项目。

#### 立项页面

```
┌─ 采购项目立项 ──────────────────────────────────────────┐
│                                                         │
│ * 项目名称：[ 输入 ]                                    │
│ * 关联计划：[ 从可执行计划中选择 ]（可多选）             │
│                                                         │
│ 已选计划：                                              │
│ - 办公楼改造（800万，工程类，拟采购方式：公开招标）      │
│ - 设备采购（1200万，工程类，拟采购方式：邀请招标）       │
│                                                         │
│ * 采购方式：[ 公开招标 ▼ ]（自动带入，可编辑）           │
│ * 估算金额：[ 2000.00 ] 万元（自动汇总）                │
│   ...（其他立项字段）                                    │
│                                                         │
│ [ 提交立项 ]                                            │
└─────────────────────────────────────────────────────────┘
```

#### 立项约束

| 约束 | 说明 |
|------|------|
| 必须关联可执行计划 | 采购项目必须关联可执行计划中的子计划/项目计划 |
| 一对一使用 | 一个子计划只能被一个采购项目使用，使用后标记为"已立项" |
| 项目类型一致性 | 多个子计划添加到同一项目时，项目类型（工程/物资/服务）必须一致，提交时校验 |
| 采购方式带入 | 子计划的拟采购方式自动带入立项表单，可编辑，最终以立项选择为准 |

#### 采购方式带入逻辑

- 选择第一个子计划后，自动带入其"拟采购方式"
- 如果选择多个子计划且拟采购方式不同，优先使用第一个子计划的拟采购方式
- 用户可修改采购方式，最终以立项选择的为准

## 五、关键业务规则

### 5.1 编制规则

| 规则 | 说明 |
|------|------|
| 编制顺序 | 先主表 → 子计划/项目计划 → 明细 |
| 金额约束 | 子计划金额总和 ≤ 年度计划总金额（强制校验，不允许超额） |
| 计划类型 | 子计划：自行采购/集中采购；项目计划：仅自行采购 |
| 项目类型 | 工程 / 物资 / 服务 |
| 明细类型 | 工程/服务类 → 项目明细；物资类 → 物资明细 |

### 5.2 审核规则

| 规则 | 说明 |
|------|------|
| 审核入口 | 统一在"采购系统 → 任务中心"处理 |
| 年度计划审核路径 | 内部审核 → 逐级审核 → 二级单位审核 |
| 项目计划审核路径 | 内部审核 |
| 自行采购计划审核路径 | 内部审核 |
| 变更审核路径 | 内部审核 |

### 5.3 分派规则

| 规则 | 说明 |
|------|------|
| 分派范围 | 仅可分派计划类型 = "集中采购" 且已生效的子计划 |
| 执行单位范围 | 仅限当前二级单位下属的单位 |
| 分派后状态 | 分派后，子计划出现在执行单位的"本企业可执行计划"中 |

### 5.4 变更规则

| 规则 | 说明 |
|------|------|
| 变更入口 | 在"本企业可执行计划"中点击"变更" |
| 变更范围 | 仅支持明细的新增/编辑/删除，不修改主表和子计划信息 |
| 原计划不变 | 本企业年度计划中的原始记录不变 |
| 变更审核 | 变更需内部审核通过后方可生效 |
| 变更期间 | 变更审核期间，原明细继续有效 |

### 5.5 立项规则

| 规则 | 说明 |
|------|------|
| 前置条件 | 必须关联可执行计划中的子计划/项目计划 |
| 一对一使用 | 一个子计划只能被一个采购项目使用 |
| 项目类型一致性 | 同一项目的多个子计划，项目类型必须一致（提交时校验） |
| 采购方式 | 拟采购方式自动带入，可编辑，以立项为准 |
| 立项后标记 | 立项后，子计划标记为"已立项"，不可再次使用 |

## 六、Excel 导入设计

### 6.1 模板分类

按业务类型提供独立模板：

| 模板名称 | 适用场景 | 导入目标 |
|---------|---------|---------|
| 工程类计划模板 | 工程类子计划/项目计划 | 子计划 + 项目明细 |
| 货物类计划模板 | 货物类子计划/项目计划 | 子计划 + 物资明细 |
| 服务类计划模板 | 服务类子计划/项目计划 | 子计划 + 项目明细 |

**注**：电煤类计划在独立菜单页面管理，使用专用模板。

### 6.2 导入流程

```
1. 用户下载对应模板
2. 填写 Excel 数据
3. 上传 Excel 文件
4. 系统校验数据（必填项、数据格式、金额范围等）
5. 校验通过：预览导入数据，确认导入
6. 校验不通过：下载错误报告，修正后重新导入
7. 导入成功后，生成子计划/明细记录，状态为 DRAFT
```

## 七、审批流程集成

### 7.1 业务类型定义

在审批流程系统的标准业务类型库中新增：

| 业务类型编码 | 业务名称 | 适用场景 |
|------------|---------|---------|
| `PLAN_ANNUAL_APPROVAL` | 年度计划审核 | 年度计划提交审核 |
| `PLAN_PROJECT_APPROVAL` | 项目计划审核 | 项目计划提交审核 |
| `PLAN_ITEM_CHANGE` | 计划明细变更 | 已生效计划的明细变更 |

### 7.2 审批路径配置

| 业务类型 | 推荐路径 | 说明 |
|---------|---------|------|
| 年度计划审核 | 本级 → 逐级 → 二级 | 使用多级审批流，根据企业终审权限配置 |
| 项目计划审核 | 本级 | 内部审批 |
| 计划明细变更 | 本级 | 内部审批 |

### 7.3 审批实例化时机

| 业务动作 | 触发时机 | 审批实例关联字段 |
|---------|---------|----------------|
| 年度计划提交 | 点击"提交审核" | `plan_annual.approval_instance_no` |
| 项目计划提交 | 点击"提交审核" | `plan_project.approval_instance_no` |
| 明细变更提交 | 变更明细后提交审核 | 新增变更记录表关联 |

### 7.4 审批回调处理

审批流完成后，系统需回调更新业务表状态：

**审批通过**：
- 年度计划/项目计划：`plan_status = 'APPROVED'`，`edit_locked = false`
- 明细变更：更新明细数据，标记为已生效

**审批不通过**：
- 年度计划/项目计划：`plan_status = 'REJECTED'`，`edit_locked = false`
- 明细变更：变更失效，恢复原明细

## 十一、开发实施建议

### 11.1 开发顺序

| 阶段 | 任务 | 依赖 |
|------|------|------|
| Phase 1 | 数据库表结构调整（remark 字段） | 无 |
| Phase 2 | 本企业年度计划后端服务（CRUD、金额校验） | Phase 1 |
| Phase 3 | 本企业年度计划前端页面（向导式、列表、详情） | Phase 2 |
| Phase 4 | 项目计划后端服务 | Phase 1 |
| Phase 5 | 项目计划前端页面 | Phase 4 |
| Phase 6 | Excel 导入功能（模板生成、数据校验、批量导入） | Phase 2, Phase 4 |
| Phase 7 | 所属企业年度计划（查看、分派） | Phase 3, Phase 5 |
| Phase 8 | 本企业可执行计划（查看、变更） | Phase 3, Phase 5 |
| Phase 9 | 审批流程集成（业务类型注册、实例化、回调） | Phase 2, Phase 4 |
| Phase 10 | 采购项目立项（关联计划、约束校验） | Phase 8 |
| Phase 11 | 联调测试 | Phase 3-10 |

### 11.2 关键技术点

1. **多态关联处理**：`owner_type + owner_id` 的多态归属需在前端正确解析
2. **审批流集成**：需与审批模块深度集成
3. **Excel 导入**：使用 Apache POI 或 EasyExcel 处理大文件导入
4. **状态机管理**：计划状态流转建议使用状态机模式
5. **并发控制**：变更审核期间的编辑锁定需配合乐观锁
6. **金额校验**：子计划金额总和校验需在事务内完成

### 11.3 注意事项

1. 计划审核不通过时，需保留驳回原因供用户查看
2. 明细变更审批期间，原明细必须保持可用状态
3. Excel 导入需提供详细的错误报告，方便用户修正
4. 审批记录展示需复用审批流式记录视觉组件
5. 立项关联计划后，需标记子计划的"已立项"状态防止重复立项
6. 电煤计划独立页面需单独设计，字段和流程与普通计划差异较大
7. 集采分派后，执行单位可执行计划需正确显示来源为"集采分派"
