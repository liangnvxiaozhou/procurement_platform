## 一、功能概述

采购计划模块用于对集团的采购计划进行制定和管理，为企业提供及时准确的采购计划和执行路线。

本模块支持企业建立年度采购计划、变更子计划及采购项目立项；支持二级单位审核控股投资企业年度采购计划及分派集中采购计划。

### 模块划分

```
采购计划管理
│
├── 1. 本企业年度计划（编制端）
│   ├── 年度计划类型：年度计划 → 子计划（自行/集采）→ 明细
│   └── 项目计划类型：项目计划（仅自行采购）→ 明细
│
├── 2. 所属企业年度计划（查看+分派端）
│   └── 仅展示下级单位已生效的计划（年度计划/项目计划）
│   └── 支持分派集采子计划给执行单位
│
├── 3. 本企业可执行计划（执行端）
│   ├── 本企业审核通过的子计划/项目计划
│   ├── 二级单位分派的集采子计划
│   └── 支持变更明细，重新提交审核
│
└── 4. 采购项目立项（立项端）
    └── 基于可执行计划中的子计划/项目计划进行立项
    └── 拟采购方式自动带入，可编辑

审核入口：采购系统 → 任务中心（统一处理所有计划审核）
```

## 二、核心业务流程

采购计划模块围绕"编制 → 审核 → 执行 → 立项"的完整业务链路展开，支持年度计划和项目计划两种管理模式，并涵盖集中采购计划的特殊分派场景。

### 2.1 编制流程

```
年度计划：
  创建年度计划主表 → 添加子计划 → 添加明细 → 提交审核

项目计划：
  创建项目计划主表 → 添加明细 → 提交审核
```

### 2.2 审核流程（在任务中心处理）

```
年度计划：
  内部审核 → 逐级审核 → 二级单位审核 → 生效

项目计划/自行采购计划：
  内部审核 → 生效
```

### 2.3 分派流程（仅集采）

```
二级单位查看已生效集采子计划 → 选择执行单位 → 分派
```

### 2.4 执行流程

```
查看可执行计划 → 变更明细（需审核）→ 立项
```

### 2.5 立项流程

```
选择可执行计划中的子计划 → 创建采购项目

约束：
  - 一个子计划仅用一次
  - 多子计划项目类型须一致
  - 拟采购方式自动带入，可编辑
```

## 三、数据结构

### 3.1 年度计划结构

```
年度计划主表（1）
  └── 子计划（N）一对多关系
        └── 明细（N）一对多关系
              ├── 工程/服务类明细
              └── 物资类明细
```

**关系说明**：
- 1个年度计划 → 多个子计划
- 1个子计划 → 多个明细

### 3.2 年度计划主表字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 年度计划名称 | 文本 | ✅ | |
| 项目性质 | 下拉选择 | ✅ | 基建类 / 技改类 / 经营类 / 工程承包类 / 大宗物资类 |
| 预算金额 | 数字 | ✅ | 单位：万元 |
| 计划采购时间 | 年月选择器 | ✅ | yyyy-mm |
| 预计结束时间 | 年月选择器 | ✅ | yyyy-mm |
| 备注 | 文本域 | ❌ | |

#### 新增年度计划（向导式）

**步骤导航**：

```
● 步骤1：基础信息  →  ○ 步骤2：添加子计划  →  ○ 步骤3：添加明细  →  ○ 步骤4：确认提交
```

**步骤1：创建年度计划主表**

字段定义参见 [3.2 年度计划主表字段定义](#32-年度计划主表字段定义)。

**步骤2：添加子计划**

字段定义参见 [3.3 子计划字段定义](#33-子计划字段定义)。

**自动带入规则**：
- 项目类型：年度计划项目性质 = 基建/技改/经营/工程承包类 → 默认"工程"；= 大宗物资类 → 默认"物资"
- 项目性质：自动带入年度计划的项目性质

**校验规则**：
- 子计划金额总和 ≤ 年度计划总金额（强制校验，不允许超额）
- 明细估算金额之和 ≤ 子计划估算金额

**交互说明**：
- 顶部实时显示预算金额使用情况（已用金额/剩余金额）
- 添加子计划时，如果金额超额，实时提示并阻止保存
- 支持拖拽排序子计划
- 子计划以列表形式展示，支持编辑/删除操作
- 添加子计划通过弹窗/抽屉形式，包含子计划基本信息和明细列表
- 明细根据项目类型自动切换表单（工程/服务类 ↔ 物资类）

**步骤3：添加明细**

根据子计划的项目类型，展示不同的明细表单，支持手工添加 / Excel导入 / Excel导出。

字段定义参见：
- 工程/服务类：[3.4 工程/服务类明细字段](#34-工程服务类明细字段)
- 物资类：[3.5 物资类明细字段](#35-物资类明细字段)

**步骤4：确认提交**

- 预览全部信息（年度计划主表 + 子计划列表 + 明细汇总）
- 校验结果展示
- 提交审核 → 进入任务中心待审核

- 执行单位可基于此子计划进行立项

### 4.3 本企业可执行计划

**功能路径**：`采购系统 → 采购计划 → 本企业可执行计划`

**功能定位**：查看已生效的子计划/项目计划，支持变更明细。

#### 列表页

```
┌─ 本企业可执行计划 ────────────────────────────────────────┐
│                                                           │
│ 来源：[ 全部 ▼ ]（本企业/集采分派）                       │
│ 筛选：[ 计划名称 ] [ 计划类型 ▼ ] [ 搜索 ]                │
│                                                           │
│ 计划名称 | 来源 | 估算金额 | 项目类型 | 已立项 | 操作     │
│ ────────────────────────────────────────────────────────
│ 办公楼改造 | 本企业   | 800万  | 工程 | 否 | 查看/变更/立项│
│ 电脑采购   | 集采分派 | 50万   | 物资 | 否 | 查看/变更/立项│
│ 设备采购   | 本企业   |1200万  | 工程 | 是 | 查看         │
│                                                           │
└───────────────────────────────────────────────────────────┘
```

- **来源筛选**：全部 / 本企业 / 集采分派
- **列表字段**：计划名称、来源、估算金额、项目类型、已立项状态、操作
- **操作按钮**：
  - 未立项：查看 / 变更 / 立项
  - 已立项：查看

#### 变更操作

**交互流程**：

```
可执行计划列表 → 点击"变更" → 计划状态变为 CHANGED（已变更）→ 
新页面打开编辑表单 → 修改明细 → 提交审核 → 
计划状态变为 PENDING（待审核）→ 进入任务中心待审核 → 
审核通过 → 计划状态变为 APPROVED（已生效），可执行计划更新
审核不通过 → 计划状态变为 CHANGED（已变更），可继续编辑
```

**变更页面**：

- 新页面打开，展示子计划/项目计划信息（只读）
- 明细列表支持：新增 / 编辑 / 删除
- 修改后可保存（状态保持 CHANGED）或提交审核（状态变为 PENDING）
- **关键规则**：
  - 本企业年度计划中的原始记录不变
  - 变更审核通过后，可执行计划中的明细更新
  - CHANGED 和 PENDING 状态下，该计划不能用于立项
  - 变更审核期间，原已生效的明细继续有效
  - 支持"撤销变更"，将计划状态从 CHANGED 恢复为 APPROVED

```
│                                                         │
│ * 采购方式：[ 公开招标 ▼ ]（自动带入，可编辑）           │
│ * 估算金额：[ 2000.00 ] 万元（自动汇总）                │
│   ...（其他立项字段）                                    │
│                                                         │
│ [ 提交立项 ]                                            │
└─────────────────────────────────────────────────────────┘
```

#### 立项约束

| 约束 | 说明 |
|------|------|
| 必须关联可执行计划 | 采购项目必须关联可执行计划中的子计划/项目计划 |
| 一对一使用 | 一个子计划只能被一个采购项目使用，使用后标记为"已立项" |
| 项目类型一致性 | 多个子计划添加到同一项目时，项目类型（工程/物资/服务）必须一致，提交时校验 |
| 采购方式带入 | 子计划的拟采购方式自动带入立项表单，可编辑，最终以立项选择为准 |

#### 采购方式带入逻辑

- 选择第一个子计划后，自动带入其"拟采购方式"
- 如果选择多个子计划且拟采购方式不同，优先使用第一个子计划的拟采购方式
- 用户可修改采购方式，最终以立项选择的为准

## 五、关键业务规则

### 5.1 编制规则

| 规则     | 说明                                               |
| ------ | ------------------------------------------------ |
| 编制顺序   | 先主表 → 子计划 → 明细                                   |
| 金额约束1  | 子计划金额总和 ≤ 年度计划总金额（强制校验，不允许超额）                    |
| 金额约束2  | 明细估算金额之和 ≤ 子计划估算金额（强制校验）                         |
| 计划类型   | 子计划：自行采购/集中采购；项目计划：仅自行采购                         |
| 项目类型   | 工程 / 物资 / 服务                                     |
| 明细类型   | 工程/服务类 → 项目明细；物资类 → 物资明细                         |
| 自动带入   | 子计划的项目类型和项目性质根据年度计划自动带入                          |
| 自动计算   | 子计划的计划采购时间 = 明细计划采购时间最早值；拟开工/交货时间 = 明细开工/交货时间最晚值 |
| 物资金额计算 | 物资类明细估算金额 = 采购数量 × 预估单价（转换为万元）                   |

### 5.2 审核规则

| 规则 | 说明 |
|------|------|
| 审核入口 | 统一在"采购系统 → 任务中心"处理 |
| 年度计划审核路径 | 内部审核 → 逐级审核 → 二级单位审核 |
| 项目计划审核路径 | 内部审核 |
| 自行采购计划审核路径 | 内部审核 |
| 变更审核路径 | 内部审核 |

### 5.3 分派规则

| 规则 | 说明 |
|------|------|
| 分派范围 | 仅可分派计划类型 = "集中采购" 且已生效的子计划 |
| 执行单位范围 | 仅限当前二级单位下属的单位 |
| 分派后状态 | 分派后，子计划出现在执行单位的"本企业可执行计划"中 |

### 5.4 变更规则

| 规则 | 说明 |
|------|------|
| 变更入口 | 在"本企业可执行计划"中点击"变更" |
| 变更范围 | 仅支持明细的新增/编辑/删除，不修改主表和子计划信息 |
| 原计划不变 | 本企业年度计划中的原始记录不变 |
| 变更审核 | 变更需内部审核通过后方可生效 |
| 变更期间 | 变更审核期间，原已生效的明细继续有效 |
| 状态流转 | APPROVED → 变更 → CHANGED → 提交审核 → PENDING → 审核通过 → APPROVED |
| 立项限制 | CHANGED 和 PENDING 状态下，该计划不能用于立项 |
| 撤销变更 | 支持从 CHANGED 状态撤销变更，恢复为 APPROVED |

### 5.5 立项规则

| 规则 | 说明 |
|------|------|
| 前置条件 | 必须关联可执行计划中的子计划/项目计划 |
| 一对一使用 | 一个子计划只能被一个采购项目使用 |
| 项目类型一致性 | 同一项目的多个子计划，项目类型必须一致（提交时校验） |
| 采购方式 | 拟采购方式自动带入，可编辑，以立项为准 |
| 立项后标记 | 立项后，子计划标记为"已立项"，不可再次使用 |

## 七、审批流程集成

### 7.1 业务类型定义

在审批流程系统的标准业务类型库中新增：

| 业务类型编码 | 业务名称 | 适用场景 |
|------------|---------|---------|
| `PLAN_ANNUAL_APPROVAL` | 年度计划审核 | 年度计划提交审核 |
| `PLAN_PROJECT_APPROVAL` | 项目计划审核 | 项目计划提交审核 |
| `PLAN_ITEM_CHANGE` | 计划明细变更 | 已生效计划的明细变更 |

### 7.2 审批路径配置

| 业务类型 | 推荐路径 | 说明 |
|---------|---------|------|
| 年度计划审核 | 本级 → 逐级 → 二级 | 使用多级审批流，根据企业终审权限配置 |
| 项目计划审核 | 本级 | 内部审批 |
| 计划明细变更 | 本级 | 内部审批 |

### 7.3 审批实例化时机

| 业务动作 | 触发时机 | 审批实例关联字段 |
|---------|---------|----------------|
| 年度计划提交 | 点击"提交审核" | `plan_annual.approval_instance_no` |
| 项目计划提交 | 点击"提交审核" | `plan_project.approval_instance_no` |
| 明细变更提交 | 变更明细后提交审核 | 新增变更记录表关联 |

### 7.4 审批回调处理

审批流完成后，系统需回调更新业务表状态：

**审批通过**：
- 首次提交：`plan_status = 'APPROVED'`，`edit_locked = false`
- 明细变更：`plan_status = 'APPROVED'`，更新明细数据，标记为已生效

**审批不通过**：
- 首次提交：`plan_status = 'REJECTED'`，`edit_locked = false`，记录驳回原因
- 明细变更：`plan_status = 'CHANGED'`，变更失效，恢复原明细，可继续编辑

## 十、状态字典

### 10.1 计划状态

| 状态码        | 状态名   | 可编辑 | 可立项 | 说明       |
| ---------- | ----- | --- | --- | -------- |
| `DRAFT`    | 草稿    | ✅  | ❌  | 编制中      |
| `PENDING`  | 待审核   | ❌  | ❌  | 已提交审批，等待审核 |
| `APPROVED` | 已生效   | ❌  | ✅  | 审批通过，可执行，可立项 |
| `REJECTED` | 审核不通过 | ✅  | ❌  | 审批未通过，可编辑后重新提交 |
| `CHANGED`  | 已变更   | ✅  | ❌  | 已发起变更，可编辑明细，不能立项 |

**状态流转图**：

```
首次编制：
  DRAFT → 提交审核 → PENDING → 审核通过 → APPROVED
                        ↘ 审核不通过 → REJECTED → 编辑 → DRAFT

变更流程：
  APPROVED → 点击变更 → CHANGED → 提交审核 → PENDING → 审核通过 → APPROVED
                          ↘ 撤销变更         ↘ 审核不通过 → CHANGED
```

## 十一、开发实施建议

### 11.1 开发顺序

| 阶段 | 任务 | 依赖 |
|------|------|------|
| Phase 1 | 数据库表结构调整（remark 字段） | 无 |
| Phase 2 | 本企业年度计划后端服务（CRUD、金额校验） | Phase 1 |
| Phase 3 | 本企业年度计划前端页面（向导式、列表、详情） | Phase 2 |
| Phase 4 | 项目计划后端服务 | Phase 1 |
| Phase 5 | 项目计划前端页面 | Phase 4 |
| Phase 6 | Excel 导入功能（模板生成、数据校验、批量导入） | Phase 2, Phase 4 |
| Phase 7 | 所属企业年度计划（查看、分派） | Phase 3, Phase 5 |
| Phase 8 | 本企业可执行计划（查看、变更） | Phase 3, Phase 5 |
| Phase 9 | 审批流程集成（业务类型注册、实例化、回调） | Phase 2, Phase 4 |
| Phase 10 | 采购项目立项（关联计划、约束校验） | Phase 8 |
| Phase 11 | 联调测试 | Phase 3-10 |

### 11.2 关键技术点

1. **多态关联处理**：`owner_type + owner_id` 的多态归属需在前端正确解析
2. **审批流集成**：需与审批模块深度集成
3. **Excel 导入**：使用 Apache POI 或 EasyExcel 处理大文件导入
4. **状态机管理**：计划状态流转建议使用状态机模式
5. **并发控制**：变更审核期间的编辑锁定需配合乐观锁
6. **金额校验**：子计划金额总和校验需在事务内完成

### 11.3 注意事项

1. 计划审核不通过时，需保留驳回原因供用户查看
2. 明细变更审批期间，原明细必须保持可用状态
3. Excel 导入需提供详细的错误报告，方便用户修正
4. 审批记录展示需复用审批流式记录视觉组件
5. 立项关联计划后，需标记子计划的"已立项"状态防止重复立项
6. 电煤计划独立页面需单独设计，字段和流程与普通计划差异较大
7. 集采分派后，执行单位可执行计划需正确显示来源为"集采分派"
