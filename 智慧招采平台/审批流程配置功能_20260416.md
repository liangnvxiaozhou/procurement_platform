## 一、功能概述

为多级组织（五级、四级、三级、二级、集团等）提供统一的审批模板配置能力。支持：
- 各单位配置本组织的审批模板（最多3个节点，设置每个节点的候选人规则）。
- 上级单位查看下级单位的审批模板，并设置下级单位的“能否终审”开关（即 `allow_final_approve` / `require_parent_review`）。
- 上级单位如需承接下级上送的审批，可配置本单位自己的模板节点。

> 注：本需求不涉及免审直通（`skip_approval`），暂不实现。

## 二、页面结构

### 2.1 审批模板列表页

**路由**：`/approval/template/list`  
**权限**：登录用户需拥有“审批模板管理”权限。根据用户角色展示不同范围的数据。

#### 2.1.1 页面布局

```
┌─────────────────────────────────────────────────────────────┐
│ 审批模板管理                                                │
├──────────────┬──────────────────────────────────────────────┤
│ 组织树        │ 模板列表                                     │
│ ├─ 集团       │ ┌─────────────────────────────────────────┐ │
│ ├─ 二级公司A  │ │ 业务类型 │ 模板名称 │ 版本 │ 状态 │ 操作 │ │
│ ├─ 三级公司A1 │ │ 采购计划 │ ...     │ v1   │ 生效 │ 详情 │ │
│ └─ 四级公司A11│ └─────────────────────────────────────────┘ │
└──────────────┴──────────────────────────────────────────────┘
```

#### 2.1.2 左侧组织树

- 展示当前用户有权限查看的组织树（基于 `sys_org`）。
- 点击组织节点，右侧加载该组织的所有审批模板（按业务类型分组或列表）。
- 当前用户角色：
  - **系统管理员**：可见所有组织。
  - **上级审批管理员**：可见本级及下级组织。
  - **本单位审批管理员**：仅可见本组织。

#### 2.1.3 右侧模板列表

**筛选栏**：
- 业务类型下拉（必选，支持搜索）
- 模板状态（全部/生效中/历史版本）

**列表字段**：

| 字段 | 说明 |
|------|------|
| 业务类型 | 如“采购计划修改” |
| 模板名称 | 如“五级公司采购计划修改模板” |
| 版本号 | v1, v2 ... |
| 状态 | 当前生效 / 历史版本 |
| 最后修改时间 | - |
| 操作 | 详情 / 编辑（仅可编辑当前生效版本，且根据权限决定可编辑字段） |

**按钮**：
- “新建模板”：为当前选中组织创建新模板（同一业务类型同一组织只能有一个当前生效版本，新建后旧版本自动变为历史）。

#### 2.1.4 特殊提示

- 如果某个组织缺少某业务类型的模板，在列表中显示“未配置”，并提供“去配置”链接。

---

### 2.2 审批模板详情/编辑页

**路由**：`/approval/template/detail?id=xxx` 或 `/approval/template/edit?id=xxx`  
**权限控制**：根据当前用户角色和模板所属组织，决定可编辑字段。

#### 2.2.1 页面分为三个区域

1. **基本信息区**（部分字段根据权限只读）
2. **节点列表区**（本单位可编辑节点，上级只能查看下级节点的配置）
3. **操作按钮**（保存、取消、发布版本）

#### 2.2.2 基本信息字段

| 字段名    | 对应数据库字段                 | 控件类型   | 必填  | 权限说明                           |
| ------ | ----------------------- | ------ | --- | ------------------------------ |
| 业务类型   | `business_type`         | 下拉（只读） | 是   | 创建时选定，不可修改                     |
| 适用组织   | `org_id`                | 组织树只读  | 是   | 创建时选定，不可修改                     |
| 模板名称   | `def_name`              | 输入框    | 是   | 本单位可编辑；上级查看下级时只读               |
| 模板编码   | `def_code`              | 输入框    | 是   | 同上                             |
| 允许终审   | `allow_final_approve`   | 开关     | 否   | **仅上级单位可编辑**（修改下级模板时）；本单位查看时只读 |
| 要求上级审批 | `require_parent_review` | 开关     | 否   | **仅上级单位可编辑**（修改下级模板时）；本单位查看时只读 |
| 备注     | `remark`                | 文本域    | 否   | 本单位可编辑；上级只读                    |

> **联动规则**（前端校验）：
> - `require_parent_review = true` 时，`allow_final_approve` 必须为 `false`。
> - `require_parent_review = false` 时，`allow_final_approve` 必须为 `true`。
> - 保存时如果违反规则，提示“不允许同时为false/true，请调整”。

#### 2.2.3 节点列表区

**展示方式**：卡片列表或表格，按 `step_no` 顺序排列。

**每个节点包含以下字段**：

| 字段名 | 对应字段 | 控件 | 必填 | 说明 |
|--------|----------|------|------|------|
| 节点顺序 | `step_no` | 自动生成，支持拖拽调整 | 是 | 1,2,3... |
| 节点名称 | `step_name` | 输入框 | 是 | 如“部门负责人审批” |
| 节点编码 | `step_code` | 输入框 | 是 | 唯一标识 |
| 候选类型 | `candidate_type` | 下拉 | 是 | 选项：角色(ROLE)、用户(USER)、岗位(POST) |
| 候选值 | `candidate_value` | 根据候选类型动态控件 | 是 | 角色选择器/用户选择器/岗位选择器（支持多选） |
| 分配模式 | `assignment_mode` | 下拉 | 是 | `SELECT_ONE`（指定一人）、`ANY_ONE`（任意一人） |
| 是否可显式指定下一审批人 | `selectable_next_assignee` | 开关 | 否 | 默认 `true`，仅 `SELECT_ONE` 时有效 |
| 是否允许撤回 | `allow_withdraw_enabled` | 开关 | 否 | 默认 `false` |
| 拒绝处理类型 | `reject_to_type` | 只读 | 否 | 固定为 `END_INSTANCE` |

**操作**：
- 添加节点（最多3个，超出禁用按钮并提示）
- 删除节点（至少保留1个节点，删除前确认）
- 上移/下移（调整顺序）

**权限**：
- **本单位编辑自己模板**：可增删改节点，所有节点字段可编辑。
- **上级查看下级模板**：节点列表只读，不可修改。

#### 2.2.4 版本管理提示

- 每次编辑保存后，生成一个新版本（`version_no` 递增），并将新版本设为 `is_current = true`，旧版本 `is_current = false`。
- 已发起的审批实例继续使用旧版本快照，不受影响。
- 页面需显示当前编辑的是哪个版本（如“当前生效版本 v2”），并提供“查看历史版本”入口（弹窗列表）。

#### 2.2.5 按钮

- **保存**：校验通过后，生成新版本（若无改动则提示无变化）。
- **取消**：返回列表页，不保存。
- **发布**（可选）：如果设计了“草稿”状态，则需要发布才生效；为了简化，可以直接保存即生效（生成新版本）。建议采用保存即生效。

## 三、关键交互与校验规则

### 3.1 新建模板流程

1. 在列表页选择组织、业务类型，点击“新建模板”。
2. 弹出或跳转到详情页，基本信息预填：业务类型、组织（不可改）。
3. 模板名称、模板编码需用户填写。
4. 节点列表默认展示一个空节点（或给一个默认示例节点），用户可增删。
5. 保存时校验节点数量 ≤ 3，且至少1个节点。
6. 保存成功后返回列表页，新模板出现在列表中，状态为“当前生效”。

### 3.2 编辑模板流程

1. 在列表页点击某条模板的“编辑”按钮。
2. 加载当前生效版本的数据（如果编辑的是历史版本，提示“历史版本不可编辑，请复制创建新版本”）。
3. 根据用户权限，展示可编辑字段：
   - 本单位管理员：可编辑节点列表，基本信息中的终审/上送开关只读。
   - 上级管理员（查看下级模板）：可编辑终审/上送开关，节点列表只读；若上级需要修改下级节点？根据业务描述，上级不应修改下级节点，只读。
   - 上级管理员编辑自己本组织的模板：与本单位权限一致（可编辑节点）。
4. 保存时校验，生成新版本。

### 3.3 删除模板

- 当前生效版本不可删除（可禁用或归档）。
- 历史版本可以删除，但需检查是否有审批实例引用（若有则不可删）。
- 提供“停用”功能：将 `is_current` 设为 false，模板不再用于新发起审批。

### 3.4 候选值选择器

- **角色选择器**：从 `sys_role` 中获取角色列表，支持多选，存储角色编码（逗号分隔或 JSON）。
- **用户选择器**：从 `sys_user` 中获取用户，支持多选，存储用户 UID。
- **岗位选择器**：从预定义的岗位字典中获取，支持多选，存储岗位编码。

### 3.5 候选人运行时解析说明

在配置界面，每个节点可提供“预览候选人”按钮，根据当前组织、候选规则，实时调用后端接口解析出候选人列表，帮助管理员验证配置是否正确。此功能可选。

## 五、接口需求（供后端参考）

前端需要以下 API：

1. **获取组织树**（带权限过滤）
2. **获取某组织下某业务类型的模板列表**（支持版本筛选）
3. **获取模板详情**（包含节点列表）
4. **创建模板**（保存基本信息+节点）
5. **更新模板**（生成新版本）
6. **删除模板**（仅历史版本）
7. **预览候选人**（根据候选规则解析用户列表）
8. **校验审批链完整性**（可选，用于上级配置终审开关时检查下级及上上级模板是否存在）

## 六、异常提示文案示例

- “该组织下已存在当前生效模板，新建后旧版本将自动归档。”
- “节点数量不能超过3个。”
- “要求上级审批为是时，允许终审必须为否。”
- “缺少上级组织【四级公司】的【采购计划修改】审批模板，请先配置。”
- “历史版本不可编辑，如需修改请基于此版本创建新模板。”
