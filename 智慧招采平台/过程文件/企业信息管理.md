
### 运营后台侧边导航菜单设计 (Sidebar Menu)

> **[System Instructions]**
> 
> Generate a vertical sidebar navigation menu for the "Procurement Operations Backend" (运营后台).
> 
> **CRITICAL:** All menu titles and sub-menus must be in **Simplified Chinese (简体中文)**.
> 
> **[Styling Guidelines]**
> 
> - **Primary Brand Color**: `#0A62CA` for active/selected states.
>     
> - **Hover State**: `#4080FF`.
>     
> - **Typography**: 14px base font size, Medium weight for top-level menus.
>     
> - **Visual Style**: Professional enterprise B-side UI.
>     
> 
> **[Updated Menu Structure & Icons]**
> 
> 1. **租户管理 (Tenant Management)**: Icon - Building/Unit.
>     
> 2. **权限管理 (Permission Management)**: Icon - Lock/Shield.
>     
> 3. **用户管理 (User Management)**: Icon - Users/Avatar. (Expandable)
>     
>     - **企业管理 (Enterprise Management)**: (Note: Contains the Tab-based list page previously designed).
>         
>     - **企业信息变更审核 (Enterprise Info Change Audit)**: Focus on auditing modified enterprise profiles.
>         
>     - **账号申诉审核 (Account Appeal Audit)**: Handle lost admin account recovery requests.
>         
> 4. **供应商管理 (Supplier Management)**: Icon - Truck/Solution.
>     
> 5. **采购管理 (Procurement Management)**: Icon - Monitor/FileDone.
>     
> 6. **运营管理 (Operations Management)**: Icon - Tool/Notification.
>     
> 7. **全局日志 (Global Audit Logs)**: Icon - History/Search.
>     
> 8. **系统配置 (System Configuration)**: Icon - Settings/Gear.
>     


> 
> **[Page Structure: List Page]**
> 
> 1. **Header**: Title "企业管理".
>     
> 2. **Global Actions (全局操作)**: Primary buttons for "批量审核" (Batch Audit) and "批量导出" (Batch Export).
>     
> 3. **Search Bar**: Filters for "企业名称", "社会信用代码", "联系人", "注册时间" (Date Range), and "所属租户" (Dropdown).
>     
> 4. **Tabs Implementation**:
>     
>     - **待审核 (To be Audited)**:
>         
>         - Columns: 企业名称, 社会信用代码, 交易市场主体, 用户名称, 注册手机号, 联系人, 注册时间.
>             
>         - Actions: 查看详情, 审核通过, 审核不通过.
>             
>     - **审核通过 (Approved)**:
>         
>         - Columns: [Same as above] + 审核时间, 审核员.
>             
>         - Actions: 查看详情, 审核不通过, 重置密码, 设为无效.
>             
>     - **审核不通过 (Rejected)**:
>         
>         - Columns: [Same as above] + 审核时间, 审核员, 不通过原因.
>             
>         - Actions: 查看详情, 审核通过.
>             
>     - **注册中 (Registering)**:
>         
>         - Columns: 企业名称, 社会信用代码, 交易市场主体, 用户名称, 注册手机号, 联系人, 注册时间.
>             
>         - Actions: 查看详情.
>             
> 
> **[Page Structure: Detail Page (New Page)]**
> 
> Design a structured detail page divided into the following modules:
> 
> 1. **基础信息 (Basic Info)**:
>     
>     - **Required (*)**: 企业名称, 交易主体类型 (Multi-select: 投标人/供应商, 招标人/采购人, 招标代理), 注册邀请码, 营业执照附件 (Upload), 营业范围, 注册资金 (单位: 万元 人民币), 行业分类, 所在行政区域, 通讯地址, 注册地址, 联系人, 联系电话, 是否高新企业 (Radio), 是否小微企业 (Radio).
>         
>     - **Optional**: 企业电话, 企业传真, 职工人数, 技术人员人数, 备注.
>         
> 2. **法人代表 (Legal Representative)**:
>     
>     - **Required (*)**: 姓名, 证件类型 (Select: 居民身份证, 护照/其他), 证件号码, 证件附件 (Upload).
>         
> 3. **发票信息 (Invoicing Info)**:
>     
>     - **Required (*)**: 是否一般纳税人 (Radio), 纳税人识别码 (Read-only), 地址, 电话, 开户行, 开户行账号.
>         
> 4. **资质信息 (Qualification)**:
>     
>     - A dynamic list/table with "添加", "编辑", "删除" functions.
>         
>     - **Required fields per item**: 资质证书名称, 资质证书编号, 资质证书级别, 资质证书附件 (Upload), 有效期开始时间, 有效期结束时间.
>         
> 
> **[Interaction & Logic]**
> 
> - **Navigation**: Clicking "查看详情" opens the detail page in a new tab.
>     
> - **Audit Action**: Clicking "审核不通过" must trigger a dialog asking for "不通过原因" (Reason).
>     
> - **Invalidation**: "设为无效" requires a secondary confirmation pop-up.
>     
> - **Consistency**: Use `#0A62CA` for all primary buttons and active Tab underlines.
>
