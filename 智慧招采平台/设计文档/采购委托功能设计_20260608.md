> 设计日期：2026-06-08
> 最后修改：2026-06-08 15:00:00
> 状态：需求梳理完成
> 说明：本文档定义采购委托模块的业务流程、数据结构、页面设计及与上下游模块的交互关系

## 一、功能概述

采购委托模块位于采购立项与项目管理之间，负责委托单的分配、接收、退回以及委托合同的管理。承接来自采购立项的线上委托单和代理机构代建的线下委托单，最终将符合条件的委托单交付到项目建项环节。

### 两条委托路径

| 路径 | 委托方 | 被委托方 | 流程起点 |
|------|--------|---------|---------|
| **线上委托** | 采购单位创建采购立项，主动委托代理 | 代理单位（弹窗选择） | 采购立项审批通过 → 系统自动写入指派委托单列表 |
| **线下委托** | 采购单位线下委托 | 代理单位（当前用户单位）自建单据 | 代理项目经理新建委托单 → 提交审核 → 审批通过后自动指派给创建人 |

### 模块划分

```
采购委托管理
│
├── 1. 指派委托单      ← 代理机构负责人将线上委托单指派给项目经理
│
├── 2. 接收委托单      ← 项目经理接收/退回委托单，接收后可建项
│
├── 3. 新建委托单      ← 线下委托场景，代理机构代建委托单（含立项信息）
│
└── 4. 委托合同        ← 创建、上传与采购单位签订的委托合同，建项时引用

审核入口：采购系统 → 任务中心（统一处理新建委托单、委托合同的审核）
```

### 操作角色

| 角色 | 职责 | 系统角色 |
|------|------|---------|
| 代理机构负责人 / 项目指派员 | 指派委托单、批量指派、撤回指派 | 待定（暂用 `AGENT_MANAGER`） |
| 代理项目经理 | 接收/退回委托单、新建委托单、管理委托合同、建项 | `BUSINESS_USER` |

## 二、核心业务流程

### 2.1 线上委托流程

```
采购立项（组织形式=委托采购）→ 审批通过 → 已生效
    │
    │ 系统自动将委托单数据写入「指派委托单」列表（状态：未指派）
    │
    ▼
代理机构负责人在「指派委托单」→ 选择委托单 → 指派给项目经理（一对一）
    │
    │ 采购立项状态：已生效 → 已接收
    │
    ▼  委托单状态：未指派 → 未接收
项目经理在「接收委托单」→ 查看指派的委托单
    ├── 接收 → 委托单状态：未接收 → 已指派，可建项
    │   ├── 方式A：点击「建项」→ 立项信息带入 → 跳转项目详情页（编辑模式）
    │   └── 方式B：在「我的项目」点击「新建」→ 选择「选择委托单」Tab
    └── 退回 → 委托单状态：未接收 → 未接收（回到原状态，可重新指派）
        │
        │ 采购立项状态：已接收 → 已生效（回退，可重新指派）
```

### 2.2 线下委托流程

```
代理项目经理在「新建委托单」→ 填写立项信息 + 委托单信息 → 提交审核
    │
    ▼  审批通过
系统自动指派给创建人（创建人即项目经理）→ 状态：未接收
    │
    │ 后续操作同线上流程后半段，在「接收委托单」模块操作
    │
    ▼
创建人手动点击「接收」→ 状态变为已指派 → 建项
```

**注**：线下委托单审批通过后自动指派给创建人，不经过「指派委托单」模块人工指派。

### 2.3 委托合同流程

```
代理项目经理在「委托合同」→ 新建 → 填写合同信息 + 上传合同文件 → 提交审核
    │
    ▼  本级审批通过 → 已生效
    │
    ▼  建项时手动关联引用（委托单或项目详情中弹窗选择委托合同）
```

## 三、数据结构

### 3.1 委托单主表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 委托单名称 | 文本 | ✅ | 线上：自动回显「计划名称 + 委托单」，可编辑；线下：手动填写 |
| 委托单编号 | 文本 | - | 系统自动生成，规则待定 |
| 委托来源 | 文本 | - | 系统自动：线上委托 / 线下委托，只读 |
| 关联采购立项 | 文本 | - | **线上**：关联的采购立项名称及编号，只读；**线下**：创建时填写的项目名称及项目信息摘要 |
| 采购单位 | 文本/弹窗选择 | ✅ | **线上**：只读，自动填入采购立项创建单位名称；**线下**：弹窗选择采购单位 |
| 代理单位 | 文本/弹窗选择 | ✅ | **线上**：弹窗选择代理机构；**线下**：只读，自动填入当前用户单位名称 |
| 联系人姓名 | 文本 | ✅ | 自动读取采购单位管理员姓名，可编辑 |
| 联系电话 | 文本 | ✅ | 自动读取采购单位联系电话，可编辑 |
| 关联委托合同 | 弹窗选择 | ❌ | 从已生效的委托合同中选择引用 |
| 备注 | 长文本 | ❌ | |
| 采购委托单文件 | 文件 | - | 线上：系统自动生成；线下：手动上传 |
| 附件 | 文件上传 | ❌ | 支持多个，excel/pdf/word |
| 当前指派人 | 文本 | - | 冗余字段，最近一次指派操作的目标人姓名（子表有完整记录） |
| 当前指派时间 | 日期时间 | - | 冗余字段，最近一次指派操作的时间 |
| 创建人 | 文本 | 系统自动 | |
| 创建时间 | 日期时间 | 系统自动 | yyyy-mm-dd HH:mm:ss |
| 更新人 | 文本 | 系统自动 | |
| 更新时间 | 日期时间 | 系统自动 | yyyy-mm-dd HH:mm:ss |

**线下委托额外字段（立项信息区域，仅线下委托需要）**：

线下委托单由于不经过采购立项环节，需额外承载立项信息字段。字段直接引用《采购立项功能设计》第 3.1.1 节"立项主表字段定义"，主要包括：

| 字段分类 | 包含字段 |
|---------|---------|
| 项目基础信息 | 项目名称、项目编号、项目类型、估算金额 |
| 实施信息 | 项目所属区域、项目实施地点、项目性质 |
| 采购配置 | 采购方式、资审方式、邀请供应商、询比方式、谈判方式、直接采购属性、竞价方式（及其联动规则） |
| 分类与要求 | 主要标的类型、主要标的类别占比、本项目行业分类 |
| 文件与描述 | 项目审批/核准/备案文件、项目概况 |
| 供应商要求 | 供应商基本要求、供应商资质要求、供应商业绩要求、供应商其他要求 |
| 其他 | 是否协同项目、协同目录、立项备注、附件 |

**说明**：
- 线下委托单不含"组织形式"字段（固定为委托采购）
- 不含"立项联系人/立项联系电话/立项提报单位"（已通过委托单主表的联系人姓名/联系电话/采购单位覆盖）
- 不含委托单自身字段（避免自引用）

### 3.2 指派操作日志子表

记录每次指派 / 撤回指派 / 退回操作，支持完整操作履历追溯。主表保留当前值的冗余字段供列表高频展示。

| 字段 | 类型 | 说明 |
|------|------|------|
| 序号 | 自增主键 | |
| 委托单ID | 外键 | 关联委托单主表 |
| 操作类型 | 枚举 | 指派 / 撤回指派 / 退回 |
| 操作人 | 文本 | 执行该操作的用户姓名（指派人或项目经理） |
| 操作时间 | 日期时间 | |
| 目标人 | 文本 | 指派时=被指派的项目经理；退回时=退回人（即项目经理） |
| 备注 | 长文本 | 退回时可为空（当前设计退回无需填写原因） |

**设计理由**：线上委托单存在「指派→退回→重新指派」循环，子表保留完整操作履历，主表冗余字段保证列表查询时无需JOIN。

### 3.3 委托单状态字典

统一采用单一状态字段，各模块通过 Tab 过滤展示各自视角。

**主状态值（适用于主表 `status` 字段）**：

| 状态 | 状态码 | 触发条件 | 说明 |
|------|------|---------|------|
| 草稿 | `DRAFT` | 仅线下：编辑保存 | 线下委托单编制中 |
| 待审核 | `PENDING_APPROVAL` | 仅线下：提交审核 | 线下委托单审核中 |
| 审核不通过 | `APPROVAL_REJECTED` | 仅线下：审核拒绝 | 可重新编辑提交 |
| 未指派 | `UNASSIGNED` | 仅线上：采购立项审批通过 | 等待代理机构指派项目经理 |
| 未接收 | `UNRECEIVED` | 指派后/退回后/线下审批通过后 | 等待项目经理接收 |
| 已指派 | `RECEIVED` | 项目经理接收 | 已接收，可建项 |
| 已建项 | `PROJECT_CREATED` | 项目经理建项 | 已关联到采购项目，最终状态 |

**状态流转图**：

```
线上委托单（由采购立项自动写入）：
  未指派 → 指派 → 未接收 → 接收 → 已指派 → 建项 → 已建项
                        ↑ 退回 ──┘
  撤回指派：未接收 → 未指派

线下委托单（由新建委托单审核后生成）：
  草稿 → 提交审核 → 待审核 → 审核通过 → [自动指派给创建人] → 未接收
                        ↓ 审核不通过 → [可重新编辑]
  后续同线上：未接收 → 接收 → 已指派 → 建项 → 已建项
```

### 3.4 各模块 Tab 过滤方案

| 模块 | 数据范围 | Tab 页签及其过滤条件 |
|------|---------|------------------|
| **指派委托单** | 仅线上委托单 | 全部 / 未指派 / 未接收 / 已指派 / 已建项 |
| **新建委托单** | 仅线下委托单 | 全部 / 草稿 / 待审核 / 已指派 / 已建项 |
| **接收委托单** | 当前用户的所有委托单（线上+线下） | 全部 / 未接收 / 已指派 / 已建项 |

**说明**：
- 「新建委托单」的 Tab 中，审核不通过的记录在「待审核」Tab 中以特殊标记展示，便于重新编辑
- 「接收委托单」展示当前登录用户（项目经理）相关的委托单，包括被指派和被退回的
- 「指派委托单」的"未接收"Tab 中同时包含首次指派和退回后的委托单，如需区分可通过指派操作日志查看退回标记

### 3.5 采购立项状态联动

委托单操作会联动影响采购立项的状态：

| 委托单操作 | 委托单状态变化 | 采购立项状态变化 |
|-----------|-------------|----------------|
| 指派 | 未指派 → 未接收 | 已生效 → 已接收 |
| 撤回指派 | 未接收 → 未指派 | 已接收 → 已生效 |
| 退回 | 未接收 → 未接收（重置） | 已接收 → 已生效 |
| 接收 | 未接收 → 已指派 | —（不变） |
| 建项 | 已指派 → 已建项 | 已接收 → 已建项 |

**说明**：退回后采购立项状态回退到「已生效」，委托单回到「未接收」，可被重新指派。

## 六、与现有模块的交互

### 6.1 采购立项 → 指派委托单（数据写入）

采购立项（组织形式=委托采购）审批通过后，系统自动将委托单数据写入「指派委托单」列表：
- 委托单主表的各字段从采购立项的委托单区域自动带入
- 初始状态：未指派
- **仅线上委托单写入**，线下委托单由「新建委托单」模块独立管理

### 6.2 采购立项状态联动

| 委托单操作 | 委托单状态 | 采购立项状态 |
|-----------|-----------|------------|
| 指派 | 未指派 → 未接收 | 已生效 → 已接收 |
| 撤回指派 | 未接收 → 未指派 | 已接收 → 已生效 |
| 退回 | 未接收 → 未接收 | 已接收 → 已生效 |
| 接收 | 未接收 → 已指派 | —（不变） |
| 建项 | 已指派 → 已建项 | 已接收 → 已建项 |

### 6.3 采购委托 → 项目管理（建项入口）

**方式A**：「接收委托单」列表点击「建项」→ 自动创建项目 → 跳转项目详情页（编辑模式）

**方式B**：「我的项目」点击「新建」→ 弹窗切换至「选择委托单」Tab → 选中后自动创建项目

两种方式均携带委托单立项信息、关联委托合同信息进入项目详情页。

### 6.4 委托合同引用

委托合同通过委托单主表的「关联委托合同」字段实现与委托单的关联，也可在项目建项时通过项目详情关联。均为手动弹窗选择，不自动绑定。

## 七、审批流程集成

### 7.1 审批路径

| 业务名称 | 审批路径 | 说明 |
|---------|---------|------|
| 新建委托单（线下） | 本级审批 | 代理机构内部审批，审批通过后自动指派给创建人 |
| 委托合同 | 本级审批 | 代理机构内部审批 |

### 7.2 不触发审批的操作
