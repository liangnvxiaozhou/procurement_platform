# 我的工作台

> 关联文档：[项目执行总览](README.md)

## 1. 业务流程

### 1.1 工作台定位与数据汇聚

我的工作台是执行相关角色（项目经理 / 项目成员 / 代理项目经理 / 审核人 / 监管）的聚合入口，本身不产生业务数据，数据来源于：

- **项目主表**：状态 = 已确认的项目（已确认即可执行，不引入额外执行状态机）
- **执行各子模块**：公告 / 采购文件 / 标前准备 / 开标 / 评标 / 定标 / 标后 的事件与审批流
- **上游委托单状态**：委托场景下委托单的指派 / 建项状态

工作台按当前登录用户的角色与参与关系，聚合为三类信息：

| 聚合区 | 数据来源 | 说明 |
| --- | --- | --- |
| 待办 | 各子模块审批/操作节点 | 当前用户作为处理人的任务 |
| 进行中项目 | 项目主表（已确认） | 当前用户参与的项目 |
| 统计/预警 | 待办 + 项目派生计数 | 如本周到期、逾期预警 |

### 1.2 「进入项目」定位规则

从工作台「进入项目」有两个来源，定位规则不同：

| 来源 | 定位规则 |
| --- | --- |
| 待办列表「进入项目」 | 跳转项目详情页，自动定位到该待办对应的步骤页签 |
| 进行中项目卡片「进入项目」 | 跳转项目详情页，定位到当前进度对应步骤页签（首个未完成节点）；无进度时落到「发布」 |

进入后均支持标段包切换，各标段包独立推进进度。项目详情页结构见 [01_发布管理-公告.md](01_发布管理-公告.md) 3.1。

## 2. 数据结构 + 状态值

### 2.1 待办数据结构

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 待办ID | 文本 | 系统生成 |
| 来源环节 | 枚举 | 公告 / 采购文件 / 标前准备 / 开标 / 评标 / 定标 / 标后 |
| 待办类型 | 枚举 | 如：公告待审核、公告待发布、评标待评审、定标待确认 |
| 关联项目 | 文本 | 项目名称 / 编号 |
| 关联标段包 | 文本 | 标段包名称（可多选） |
| 触发时间 | 日期时间 | 待办生成时间 |
| 处理人角色 | 枚举 | 项目经理 / 项目成员 / 审核人 / 代理项目经理 等 |
| 跳转目标 | 文本 | 项目详情页 + 步骤页签 |
| 待办状态 | 枚举 | 待处理 / 已处理 / 已逾期 |
| 截止时间 | 日期时间 | 用于逾期预警 |

### 2.2 待办来源映射表

| 来源环节 | 待办类型 | 触发条件 | 处理人角色 | 跳转步骤页签 |
| --- | --- | --- | --- | --- |
| 公告 | 公告待审核 | 公告提交审核 | 审核人 | 发布 |
| 公告 | 公告待发布 | 审核通过 + 到达发布时间 | 发布人 | 发布 |
| 采购文件 | 文件待审核 | 文件提交审核 | 审核人 | 采购文件 |
| 采购文件 | 文件待发布 | 文件审核通过 | 发布人 | 采购文件 |
| 标前准备 | 开启待开始 | 到达开启时间 | 项目经理 | 标前准备 / 开标 |
| 评标 | 评审待进行 | 进入评审阶段 | 评标专家 / 项目经理 | 评标 |
| 定标 | 定标待确认 | 评审完成 | 项目经理 / 定标决策人 | 定标 |
| 标后 | 合同待签订 | 成交结果确认 | 项目经理 | 标后 |

> 注：具体待办类型随各子模块设计完成后再补全，此处先列出已知环节。

### 2.3 角色可见性矩阵

| 角色 | 可见项目范围 | 可见待办范围 | 进入项目权限 |
| --- | --- | --- | --- |
| 项目经理（自采） | 自己建项的项目 | 自己作为处理人的待办 | ✅ 可进入并操作 |
| 项目成员 | 参与的项目 | 协作待办 | ✅ 可进入并操作（等同项目经理） |
| 代理项目经理 | 指派给自己的委托项目 | 代理操作待办 | ✅ 可进入并操作 |
| 审核人 | 不直接可见项目 | 自己作为审核节点的待办 | ✅ 进入到对应步骤页签 |
| 监管 | 全部项目（只读） | 监管预警类待办 | 🔒 只读，不可操作 |

> 注：采购人（委托方）在委托场景下是否能看到已委托出去的项目，见待确认问题 5.1。
