## 一、执行阶段定位

项目执行是采购项目的核心实施阶段。当采购项目在「我的项目」中确认（状态=已确认）后，项目经理进入项目执行阶段，按照采购方式对应业务流程依次推进各环节，直至完成采购结果并进入标后/成交后管理。

```
采购计划 → 采购立项 → 项目立项（我的项目）→ 【项目执行】 → 采购完成
                                        ↑
                                   已确认状态触发执行
```

**与前序模块衔接**：

| 前序模块 | 衔接点            | 说明               |
| ---- | -------------- | ---------------- |
| 采购立项 | 项目类型、采购方式、估算金额 | 立项审批通过后可用于建项     |
| 项目立项 | 项目确认（已确认状态）    | 项目确认后进入执行阶段      |
| 采购委托 | 委托单已建项         | 委托采购场景下，代理项目经理执行 |

**执行入口**：采购系统 → 项目管理 → 我的项目 → 进入项目

## 二、采购方式与执行流程总览

### 2.1 九种采购方式执行流程对照矩阵

| #   | 采购方式     | ① 发布     | ② 文件编制  | ③ 标前准备 | ④ 开标/开启 | ⑤ 评标/评审 | ⑥ 定标/成交 | ⑦ 标后/成交后 |
| --- | -------- | -------- | ------- | ------ | ------- | ------- | ------- | -------- |
| 1   | 公开招标     | 招标公告     | 招标文件    | ✅      | 开标      | 评标      | 定标      | 标后       |
| 2   | 邀请招标     | 邀请函      | 招标文件    | ✅      | 开标      | 评标      | 定标      | 标后       |
| 3   | 询比采购（公开） | 询比公告     | 采购文件    | ✅      | 开启      | 评审      | 成交      | 成交后      |
| 4   | 询比采购（邀请） | 邀请函      | 采购文件    | ✅      | 开启      | 评审      | 成交      | 成交后      |
| 5   | 谈判采购（公开） | 谈判公告     | 采购文件    | ✅      | 开启      | 评审      | 成交      | 成交后      |
| 6   | 谈判采购（邀请） | 邀请函      | 采购文件    | ✅      | 开启      | 评审      | 成交      | 成交后      |
| 7   | 竞价（公开）   | 竞价公告     | —（合并到③） | 竞价准备   | 竞价      | —（无）    | 成交      | 成交后      |
| 8   | 竞价（邀请）   | 竞价邀请     | —（合并到③） | 竞价准备   | 竞价      | —（无）    | 成交      | 成交后      |
| 9   | 直接采购     | 事前公示→邀请函 | 采购文件    | ✅      | 开启      | 评审      | 成交      | 成交后      |

### 2.2 差异要点

| 差异维度 | 招标类          | 询比/谈判类       | 竞价类              | 直接采购         |
| ---- | ------------ | ------------ | ---------------- | ------------ |
| 发布方式 | 公开=公告；邀请=邀请函 | 公开=公告；邀请=邀请函 | 公开=公告；邀请=邀请      | 事前公示+邀请函（两步） |
| 文件名称 | 招标文件         | 采购文件         | 无独立文件阶段（合并到竞价准备） | 采购文件         |
| 开启环节 | 开标           | 开启           | 竞价（实时报价）         | 开启           |
| 评审环节 | 评标           | 评审           | 无（竞价直接出结果）       | 评审           |
| 结果环节 | 定标           | 成交           | 成交               | 成交           |

## 三、模块设计原则

- **通用优先**：先抽象通用流程和字段，再处理各采购方式的差异化需求
- **按采购方式联动**：项目关联的采购方式决定执行流程的节点编排和字段展示
- **分段推进**：每个模块独立设计，模块间通过项目状态流转衔接
- **版本可追溯**：关键环节支持版本记录和操作日志

### 4.1 模块间状态流转总图

```
项目确认（已确认）
  │
  ▼
[发布管理] ────────────────────→ 公告/邀请函已发布
  │
  ▼
[采购文件管理] ────────────────→ 文件已发布
  │
  ▼
[标前准备] ────────────────────→ 准备就绪
  │
  ▼
[开标/开启] ───────────────────→ 开标/开启完成
  │
  ▼
[评标/评审] ──（竞价无此阶段）──→ 评标/评审完成
  │
  ▼
[定标/成交] ───────────────────→ 中标/成交确认
  │
  ▼
[标后/成交后] ─────────────────→ 项目完成
```

### 4.2 通用模块抽象

| 模块 | 通用名称 | 涵盖的采购方式差异 |
|------|---------|-------------------|
| 模块1 | 发布管理（公告/邀请函） | 公开=公告，邀请=邀请函，直接采购=事前公示+邀请函 |
| 模块2 | 采购文件管理 | 招标类=招标文件，非招标类=采购文件，竞价=合并到准备阶段 |
| 模块3 | 标前准备（含报名管理） | 通用；竞价为"竞价准备" |
| 模块4 | 开标/开启 | 招标类=开标，非招标类=开启，竞价=竞价（实时报价） |
| 模块5 | 评标/评审 | 招标类=评标，非招标类=评审，竞价=无 |
| 模块6 | 定标/成交 | 招标类=定标，非招标类/竞价=成交 |
| 模块7 | 标后/成交后 | 招标类=标后，非招标类/竞价=成交后 |

## 五、角色权限（总览）

| 功能 | 角色 | 权限说明 |
|------|------|---------|
| 公告编制 | `BUSINESS_USER` / `BUSINESS_MANAGER` | 项目经理及项目组成员 |
| 公告审核 | 审批流配置的审批人 | 根据审批流配置 |
| 公告发布 | 系统 | 审批通过后到达发布时间自动发布 |
| 补遗公告编制 | `BUSINESS_USER` / `BUSINESS_MANAGER` | 项目经理及项目组成员 |

> 委托采购场景下，代理项目经理拥有同等操作权限。其余模块权限待各模块设计时补充。
