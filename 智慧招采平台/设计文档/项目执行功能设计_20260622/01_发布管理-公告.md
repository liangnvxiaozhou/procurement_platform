# 发布管理（公告）

> 关联文档：[项目执行总览](README.md)

## 1. 业务流程

### 1.1 公告发布流程

```
项目经理新建公告 → 填写公告信息 → 保存草稿
    │
    ├── 保存草稿 → 草稿状态，可继续编辑
    │
    └── 提交审核 → 待审核 → 进入任务中心
                        │
                        ├── 审批通过 → 待发布（等待到达发布时间）
                        │                │
                        │                └── 到达发布时间 → 已发布（对外可见）
                        │
                        └── 审批不通过 → 审核不通过 → 可编辑后重新提交
```

### 1.2 公告变更流程

```
草稿 / 审核不通过 / 待发布 → 点击变更 → 编辑公告内容
    │
    ├── 保存（版本号+1，写版本历史）→ 回到草稿/待发布
    │
    └── 提交审核 → 待审核 → ...（同发布流程）
```

**变更规则**：
- 变更时保留原版本快照到版本历史子表
- 主表更新为最新版本内容，版本号自增
- 待发布状态变更后回退到草稿，需重新提交审核
- 已发布状态**不可变更**

### 1.3 补遗公告流程

```
已发布公告 → 点击「发布补遗」→ 新建补遗公告（独立记录，关联原公告）
    │
    ├── 保存草稿 → 草稿状态
    │
    └── 提交审核 → 待审核 → 审批通过 → 到达发布时间 → 已发布
                                     ↘ 审核不通过 → 可编辑后重新提交
```

**补遗规则**：
- 仅已发布状态的原公告可发布补遗
- 一条原公告可发布多次补遗，补遗序号自增
- 补遗公告拥有独立的状态流转和审批流程
- 补遗公告不可再变更和再补遗

### 1.4 公告终止流程

```
已发布 → 点击「终止」→ 二次确认 → 已终止
```

**终止规则**：
- 仅已发布状态可终止
- 终止后公告对外标记为已终止，不可恢复
- 终止不影响已发布的补遗公告

### 1.5 级联时间默认值规则

以**公告开始时间**为基准锚点，标段公告信息中各时间字段按级联关系自动计算默认值：

```
公告开始时间（基准）
    │
    ├─→ 文件获取开始时间（默认 = 公告开始时间，只能选择之后的时间）
    │       │
    │       └─→ 文件获取截止时间（默认 = 公告开始时间 + N天）
    │               │
    │               └─→ 澄清截止时间（默认 = 公告开始时间 + M天，≥ 文件获取截止时间）
    │                       │
    │                       └─→ 截标/开标时间（默认 = 公告开始时间 + K天，≥ 澄清截止时间）
```

**各采购方式默认偏移量**：

| 时间字段 | 公开招标 | 询比/谈判（公开） | 竞价（公开） |
|---------|---------|----------------|------------|
| 文件获取开始时间 | +0天 | +0天 | 无此字段 |
| 文件获取截止时间 | +5天 | +3天 | 无此字段 |
| 澄清截止时间 | +6天 | +3天 | 无此字段 |
| 截标/开标时间 | +21天 | +5天 | 无此字段 |

**校验规则**：
- 文件获取开始时间 ≥ 公告开始时间
- 文件获取截止时间 ≥ 文件获取开始时间
- 澄清截止时间 ≥ 文件获取截止时间
- 截标/开标时间 ≥ 澄清截止时间
- 以上校验在提交审核时执行，校验失败滚动定位到对应字段

补遗/澄清公告为独立公告记录，关联原公告，独立审批和发布。额外增加：

| 字段 | 类型 | 说明 |
|------|------|------|
| 关联原公告ID | 外键 | 关联原公告主表 |
| 补遗类型 | 文本 | 澄清 / 补遗，默认补遗 |

**区别**：无需区域④标段公告信息；标题默认「原公告名称+补遗公告（第N次）」；独立状态流转和审批流程；不可再变更和再补遗。

### 2.7 状态字典

**公告主表状态**：

| 状态 | 状态码 | 说明 | 允许操作 |
|------|--------|------|---------|
| 草稿 | `DRAFT` | 编制中 | 编辑、提交审核、删除 |
| 待审核 | `PENDING_APPROVAL` | 已提交，审核中 | 查看、撤回 |
| 审核不通过 | `APPROVAL_REJECTED` | 审核拒绝 | 编辑、提交审核、删除 |
| 待发布 | `APPROVED` | 审批通过，未到发布时间 | 变更（覆盖+版本记录）、查看 |
| 已发布 | `PUBLISHED` | 到达发布时间，对外可见 | 查看、发布补遗公告、终止 |
| 已终止 | `TERMINATED` | 公告终止 | 查看 |

**状态流转**：

```
首次编制：
  草稿 → 提交审核 → 待审核 → 审核通过 → 待发布 → 到达发布时间 → 已发布
                      ↘ 审核不通过 → 编辑 → 草稿
                      ↘ 撤回 → 草稿

变更（仅待发布及之前）：
  草稿/审核不通过/待发布 → 变更 → 草稿/待发布 → 提交审核 → ...

补遗（仅已发布后）：
  已发布 → 新建补遗公告 → 草稿 → 提交审核 → 已发布

终止：
  已发布 → 终止 → 已终止
```

**关键规则**：
- **不可变更边界**：公告审核通过且到达发布时间（对外可见）后，不可变更，仅可发布补遗公告
- **可变更区间**：草稿、审核不通过、待发布状态可变更，变更覆盖原内容，保留版本历史
- **待发布状态变更**：回退到草稿，需重新提交审核
- **撤回**：待审核状态可撤回，回到草稿
- **终止**：仅已发布状态可终止，终止后不可恢复

**补遗公告状态**：

| 状态 | 状态码 | 说明 | 允许操作 |
|------|--------|------|---------|
| 草稿 | `DRAFT` | 编制中 | 编辑、提交审核、删除 |
| 待审核 | `PENDING_APPROVAL` | 已提交，审核中 | 查看、撤回 |
| 审核不通过 | `APPROVAL_REJECTED` | 审核拒绝 | 编辑、提交审核、删除 |
| 已发布 | `PUBLISHED` | 已对外发布 | 查看 |

**说明**：补遗公告无「待发布」状态，审批通过后按发布时间自动发布。

## 3. 页面设计

### 3.1 公告列表页

**功能路径**：`采购系统 → 项目管理 → 我的项目 → 进入项目 → 公告`

```
┌─ 公告管理 ───────────────────────────────────────────────────────┐
│  [ 新建公告 ]                                                      │
│  ┌─ 全部 ─┬─ 草稿 ─┬─ 待审核 ─┬─ 待发布 ─┬─ 已发布 ─┐           │
│  │ 类型 | 公告标题 | 公告类型 | 状态 | 发布时间 | 版本 | 操作     │
│  │ 公告 | XX工程招标公告 | 招标公告 | 已发布 | 06-15 | v3 | 查看/补遗/终止 │
│  │ 补遗 | XX工程招标公告补遗公告（第1次）| 补遗 | 已发布 | 06-16 | - | 查看 │
│  │ 公告 | YY设备询比公告 | 询比公告 | 待发布 | 06-20 | v1 | 变更/查看 │
│  │ 公告 | ZZ服务谈判公告 | 谈判公告 | 草稿   | -    | v1 | 编辑/提交审核/删除 │
│  └────────────────────────────────────────────────────────────── │
└────────────────────────────────────────────────────────────────────┘
```

**操作按钮根据状态显示**：

| 类型 | 状态 | 允许操作 |
|------|------|---------|
| 公告 | 草稿 | 编辑、提交审核、删除 |
| 公告 | 待审核 | 查看、撤回 |
| 公告 | 审核不通过 | 编辑、提交审核、删除 |
| 公告 | 待发布 | 变更、查看 |
| 公告 | 已发布 | 查看、发布补遗、终止 |
| 公告 | 已终止 | 查看 |
| 补遗 | 草稿/待审核/审核不通过 | 同公告 |
| 补遗 | 已发布 | 查看 |

**说明**：列表同时展示公告和补遗公告，补遗公告以缩进或图标标识从属关系。一个项目仅可有一个公告（非终止状态），已终止后可新建。

```
│    │ * 截标/开标时间 [ 默认+21天，询比谈判+5天 ]                        │

│    │ * 开标地点     [ 采购官网名称 ]（可编辑）                          │
│    │ * 采购范围/供应商基本/资质/业绩/其他要求...                        │
│    │   项目负责人最低要求/备注（选填）                                   │
│                                                                        │
│  ▾ ⑤ 采购单位信息（只读）                                               │
│    采购单位名称/地址/联系人/联系电话                                     │
│                                                                        │
│  ▾ ⑥ 其他相关说明（模板文本，选填，可编辑覆盖）                          │
│    发布媒介说明/注册说明/标书款支付（仅招标）/平台使用费（仅非招标）/...  │
│                                                                        │
│  [ 保存草稿 ]  [ 提交审核 ]                                            │
└────────────────────────────────────────────────────────────────────────┘
```

**交互逻辑**：

| 操作 | 行为 |
|------|------|
| 进入新建页 | 自动带出①②⑤；③名称默认填充；④各标段时间默认填充；⑥模板按采购方式填充 |
| 修改公告开始时间 | ④中尚未手动修改过的时间字段同步更新默认值 |
| 添加/删除标段包 | ②列表增删，④页签同步增删对应标段Tab |
| 切换标段Tab | ④区域展示对应标段信息 |
| 点击保存草稿 | 校验公告名称、公告开始时间必填项，保存后状态为草稿 |
| 点击提交审核 | 校验全部必填项+级联时间规则，校验通过后进入待审核 |
| 校验失败 | 滚动定位到对应字段并提示错误 |

**按采购方式联动差异**：

| 采购方式 | ③发布媒体 | ④标段公告字段 | ⑥模板文本 |
|---------|----------|-------------|----------|
| 公开招标 | 采购官网 + 中国招投标公共服务平台 | 时间字段（+5/+6/+21天） + 通用字段 | 招标类模板 |
| 询比/谈判 | 仅采购官网 | 时间字段（+3/+3/+5天） + 通用字段 | 非招标类模板 |
| 竞价 | 仅采购官网 | 竞价参数字段（只读带入） + 通用字段 | 非招标类模板 |

### 3.3 公告详情页（查看模式）

**Tab 结构**：`公告信息 | 标段公告 | 版本历史 | 审批记录`

- **公告信息**：展示区域①②③⑤⑥全部字段
- **标段公告**：展示区域④各标段包的标段公告信息（页签切换标段）
- **版本历史**：展示版本历史子表记录，支持查看历史版本内容
- **审批记录**：接入审批流式记录组件

```
┌─ 公告详情 — [公告信息] [标段公告] [版本历史] [审批记录] ──────────┐
│  ▾ ①~⑥ 全部字段（只读展示）                                       │
│  ─────────────────────────────────────────────────────────────── │
│  草稿：        [ 编辑 ]  [ 提交审核 ]  [ 删除 ]                     │
│  待审核：      [ 撤回 ]  [ 关闭 ]                                   │
│  审核不通过：  [ 编辑 ]  [ 提交审核 ]  [ 删除 ]                     │
│  待发布：      [ 变更 ]  [ 关闭 ]                                   │
│  已发布：      [ 发布补遗 ]  [ 终止 ]  [ 关闭 ]                    │
│  已终止：      [ 关闭 ]                                             │
└────────────────────────────────────────────────────────────────────┘
```

### 3.4 变更操作

- 页面结构与新建公告一致，各字段回显当前内容
- 修改后保存：版本号+1，修改前内容写入版本历史子表，主表更新为最新内容
- 待发布状态变更后，回退到草稿，需重新提交审核
- 页面顶部提示："变更后将生成新版本（v{n+1}），原版本（v{n}）将保留在版本历史中。"

### 3.5 发布补遗操作

补遗公告复用原公告的区域①②③⑤⑥结构，无需区域④标段公告信息：

```
┌─ 新建补遗公告 ───────────────────────────────────────────────────┐
│  ▾ 关联信息（只读）：关联原公告 / 关联项目 / 采购方式 / 补遗序号   │
│  ▾ 补遗信息：补遗标题 / 公告开始时间 / 公告附件                   │
│  ▾ 其他相关说明（选填，默认带入原公告内容，可编辑覆盖）             │
│  [ 保存草稿 ]  [ 提交审核 ]                                        │
└────────────────────────────────────────────────────────────────────┘
```

**交互逻辑**：
- 补遗标题默认填充「原公告名称+补遗公告（第N次）」，可编辑
- 补遗序号自动计算（当前原公告下已有补遗次数+1）
- 其他相关说明默认带入原公告内容，可编辑覆盖
- 补遗公告的审批流程与原公告一致

### 3.6 终止确认弹窗

```
┌─ 确认终止公告 ────────────────────────────────────────────────┐
│  确认终止该公告？终止后公告将对外标记为已终止，不可恢复。          │
│  已发布的补遗公告不受影响。                                      │
│  · XX工程改造项目招标公告                                      │
│  [ 取消 ]  [ 确认终止 ]                                        │
└────────────────────────────────────────────────────────────────┘
```

## 4. 审批流程

| 业务 | 审批路径 | 说明 |
|------|---------|------|
| 公告发布 | 本级审批 → 二级单位审批 | 审批通过后进入待发布状态 |
| 公告变更 | 本级审批 → 二级单位审批 | 待发布状态变更后需重新审批 |
| 补遗公告发布 | 本级审批 → 二级单位审批 | 独立审批，与原公告审批互不影响 |

**审批通过后发布逻辑**：
- 审批通过后状态变为「待发布」
- 系统定时检查发布时间，到达发布时间后自动变为「已发布」
- 若发布时间已过（早于当前时间），审批通过后立即发布
