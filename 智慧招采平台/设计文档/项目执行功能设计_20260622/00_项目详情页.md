# 项目详情页（采购视角）

> 关联文档：[项目执行总览](README.md)

## 1. 业务流程

### 1.1 定位

项目详情页是「进入项目」后的落地页，**标段/包级**——一个标段/包一个页面，是执行全流程的容器。本页为**采购视角**（采购人 / 项目经理 / 项目成员），供应商视角的项目详情页后续另行设计。

### 1.2 入口与默认定位

| 入口 | 规则 |
| --- | --- |
| 我的工作台标段/包行「进入项目」 | 直接进入该标段/包详情页 |
| 我的项目「进入项目」 | 单标段包直接进；多标段包弹窗选择标段/包后进 |

- 进入后默认定位到**当前进度对应步骤页签**（首个未完成节点）；
- 无进度时落到「发布」。

### 2.1 布局

```
┌─ 项目详情页（采购视角） ───────────────────────────────────────────────────┐
│  XX项目 - 标段A                                       ┌──────────────────┐  │
│  ┌──────────────────────────────────────────────────┐ │    3d 5h后        │  │  右上角倒计时卡片
│  │ 标段编号: PROJ-26-001A  创建人: 张三  项目类型: 工程│ │  距开标倒计时(红)  │  │  (大号红色数字)
│  │ 采购方式: 公开招标    当前状态: 已发公告          │ │  参与交易家数: —   │  │
│  │ 开标时间: 2026-07-08 09:00                       │ └──────────────────┘  │
│  └──────────────────────────────────────────────────┘                       │
│  [采购项目信息] [查看异议] [失败情况] [发送短信] [操作概览]   ← 操作按钮行     │
│  [发布公告]  [采购文件]  [标前准备]  [开标]  [评标]  [定标]  [标后]          │  步骤页签(纯文字;未到达置灰)
│  ──────────────────────────────────────────────────────────────────────     │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  当前 tab 内容区（对应模块卡片，见 01~07）                               │ │
│  │  或「操作概览」视图（各步骤可用操作清单，见 2.4）                        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 header 字段

| 字段 | 来源 | 说明 |
| --- | --- | --- |
| 标段编号 | 标段包 | 标段/包编号（系统生成，规则待定） |
| 创建人 | 项目 | 项目经理（建项人） |
| 项目类型 | 项目 | 工程 / 物资 / 服务 |
| 采购方式 | 项目 | 公开招标 / 邀请招标 / 询比采购 / 谈判采购 / 直接采购 / 竞价 |
| 当前状态 | 标段包执行阶段派生 | 同工作台「项目状态」，枚举待补（见 2.5） |
| 开标时间 | 标段配置 | 招标/询比/谈判/直接采购取「截标/开标时间」；竞价取「竞价开始时间」 |

> 距开标倒计时与参与交易家数移至右上角卡片，见 2.3。

### 2.3 右上角倒计时与参与家数卡片

页面右上角卡片，展示「距开标倒计时」与「参与交易家数」：

- 倒计时数字放大、红色、卡片样式；
- 倒计时三态：未排期「—」/ 未到「Nd/Nh」（红色）/ 已过「已开标」；
- 倒计时取值同工作台「距开标时间」：招标/询比/谈判/直接采购取「截标/开标时间」，竞价取「竞价开始时间」。

**参与交易家数**按采购方式取对应步骤的递交/参与家数：

| 采购方式 | 取数步骤 | 含义 |
| --- | --- | --- |
| 公开招标 / 邀请招标 | 开标 | 递交投标的家数 |
| 询比采购 / 谈判采购 | 开启 / 评审 | 递交报价/响应的家数 |
| 竞价 | 竞价 | 参与竞价的家数 |
| 直接采购 | 开启 / 评审 | 响应的家数 |

- 未到对应步骤：显「—」；
- 进行中：显当前已递交/参与家数。

### 2.4 操作按钮行

header 字段区下方一行操作按钮：

| 按钮 | 说明 |
| --- | --- |
| 采购项目信息 | 打开项目级信息视图（项目主表字段 + 标段包列表，只读），复用[立项项目信息面板](../项目立项功能设计_通用版.md) |
| 查看异议 | 功能后续补充 |
| 失败情况 | 功能后续补充 |
| 发送短信 | 功能后续补充 |
| 操作概览 | 切换内容区为「静态操作地图」视图（各步骤可用操作清单）；再次点击切回原 tab 内容 |

> 「操作地图」为静态操作清单（每步骤涉及的可用操作），具体内容随 02~07 各模块设计完成后补全。

### 2.5 当前状态

字段名「当前状态」，语义为标段/包当前所处执行步骤，由各执行子模块进度派生（如招标公告 / 邀请函 / 采购文件 / 标前准备 / 开标 / 评标 / 定标 / 标后…）。具体枚举与派生规则待 02~07 子模块设计完成后补全，并与工作台「项目状态」共用同一取值。

### 2.6 步骤页签联动

页签按采购方式 × 公开性联动（对照 [README 2.1 全流程对照矩阵](README.md)）：

| 采购方式 × 公开性 | ①发布 | ②文件 | ③准备 | ④开启 | ⑤评审 | ⑥结果 | ⑦后续 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 招标-公开 | 招标公告 | 招标文件 | 标前准备 | 开标 | 评标 | 定标 | 标后 |
| 招标-邀请 | 邀请函 | 招标文件 | 标前准备 | 开标 | 评标 | 定标 | 标后 |
| 询比-公开 | 询比公告 | 采购文件 | 标前准备 | 开启 | 评审 | 成交 | 成交后 |
| 询比-邀请 | 邀请函 | 采购文件 | 标前准备 | 开启 | 评审 | 成交 | 成交后 |
| 谈判-公开 | 谈判公告 | 采购文件 | 标前准备 | 开启 | 评审 | 成交 | 成交后 |
| 谈判-邀请 | 邀请函 | 采购文件 | 标前准备 | 开启 | 评审 | 成交 | 成交后 |
| 竞价-公开 | 竞价公告 | — | 竞价准备 | 竞价 | — | 成交 | 成交后 |
| 竞价-邀请 | 竞价邀请 | — | 竞价准备 | 竞价 | — | 成交 | 成交后 |
| 直接采购 | 事前公示+邀请函 | 采购文件 | 标前准备 | 开启 | 评审 | 成交 | 成交后 |

- 竞价：不显示「文件」「评审」页签；
- 直接采购：发布页签 = 事前公示 + 邀请函（构成待确认，见 4.1）。

### 2.7 页签规则

- 纯文字页签，不标进度状态；
- 未到达的页签**禁用置灰**，不可点击；
- 当前所在页签高亮。
