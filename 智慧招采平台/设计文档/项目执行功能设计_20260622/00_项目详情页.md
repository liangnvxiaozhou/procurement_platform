# 项目详情页（采购视角）

> 关联文档：[项目执行总览](README.md)

## 1. 业务流程

### 1.1 定位

项目详情页是「进入项目」后的落地页，**标段/包级**——一个标段/包一个页面，是执行全流程的容器。本页为**采购视角**（采购人 / 项目经理 / 项目成员），供应商视角的项目详情页后续另行设计。

> 委托采购场景下，采购委托人（委托方）不在当前系统，未来由监督系统承载。

### 1.2 入口与默认定位

| 入口 | 规则 |
| --- | --- |
| 我的工作台标段/包行「进入项目」 | 直接进入该标段/包详情页 |
| 我的项目「进入项目」 | 单标段/包直接进；多标段/包弹窗选择标段/包后进（弹窗见[立项通用版 4.1](../项目立项功能设计_通用版.md)） |

- 进入后默认定位到**当前进度对应步骤页签**（首个未完成节点）；
- 无进度时落到第一个步骤。

## 2. 页面框架

### 2.1 布局

```
┌─ 项目详情页（采购视角，以直接采购为例） ────────────────────────────────┐
│  XX项目 - 标段A                                                       │  标题(项目名称-标段/包名称)
│  ┌──────────────────────────────────────────────────────────────────┐ │
│  │ header 字段区: 标段编号 / 创建人 / 项目类型 / 采购方式 / 当前状态 / 开标时间 │ │
│  └──────────────────────────────────────────────────────────────────┘ │
│  [采购项目信息] [查看异议] [失败情况] [发送短信] [操作概览]           │  操作按钮行
│  [事前公示] [邀请函] [采购文件] [标前准备] [开启] [评审] [成交] [成交后] │  步骤页签(纯文字;未到达置灰)
│  ──────────────────────────────────────────────────────────────────    │
│  ┌──────────────────────────────────────────────────────────────────┐ │
│  │ 内容区: 当前 tab 卡片(见 01~07) 或 操作概览视图(见 2.4)           │ │
│  └──────────────────────────────────────────────────────────────────┘ │
│  ※ 右上角倒计时卡片悬浮于页面右上角(见 2.3)                            │
└────────────────────────────────────────────────────────────────────────┘
```

### 2.2 header 字段

| 字段 | 来源 | 说明 |
| --- | --- | --- |
| 标段编号 | 标段包 | 标段/包编号（立项建项时生成，直接取用） |
| 创建人 | 项目 | 项目经理（建项人） |
| 项目类型 | 项目 | 工程 / 物资 / 服务 |
| 采购方式 | 项目 | 公开招标 / 邀请招标 / 询比采购 / 谈判采购 / 直接采购 / 竞价 |
| 当前状态 | 标段包执行阶段派生 | 同工作台「项目状态」，枚举待补（见 2.5） |
| 开标时间 | 标段配置 | 招标/询比/谈判/直接采购取「截标/开标时间」；竞价取「竞价开始时间」 |

> 距开标倒计时与参与交易家数移至右上角卡片，见 2.3。

### 2.3 右上角倒计时与参与家数卡片

页面右上角悬浮卡片：

```
┌──────────────────┐
│     3d 5h后       │  ← 大号红色数字
│   距开标倒计时    │
│  参与交易家数: —  │
└──────────────────┘
```

- 倒计时数字放大、红色、卡片样式；
- 倒计时三态：未排期「—」/ 未到「Nd/Nh」（红色）/ 已过「已开标」；
- 倒计时取值同工作台「距开标时间」：招标/询比/谈判/直接采购取「截标/开标时间」，竞价取「竞价开始时间」。

**参与交易家数**按采购方式取对应步骤的递交/参与家数：

| 采购方式 | 取数步骤 | 含义 |
| --- | --- | --- |
| 公开招标 / 邀请招标 | 开标 | 递交投标的家数 |
| 询比采购 / 谈判采购 | 开启 / 评审 | 递交报价/响应的家数 |
| 竞价 | 竞价 | 参与竞价的家数 |
| 直接采购 | 开启 / 评审 | 响应的家数 |

- 未到对应步骤：显「—」；
- 进行中：显当前已递交/参与家数。

### 2.4 操作按钮行

header 字段区下方一行操作按钮：

| 按钮 | 说明 |

| 查看异议 | 功能后续补充 |
| 失败情况 | 功能后续补充 |
| 发送短信 | 功能后续补充 |
| 操作概览 | 切换内容区为「静态操作地图」视图；再次点击切回原 tab 内容 |

**操作地图演示图**（以直接采购为例，具体操作按钮待 02~07 各模块设计后补全）：

│  事前公示 | [...]                                               │

### 2.5 当前状态

字段名「当前状态」，语义为标段/包当前所处执行步骤，由各执行子模块进度派生（如招标公告 / 邀请函 / 采购文件 / 标前准备 / 开标 / 评标 / 定标 / 标后…）。具体枚举与派生规则待 02~07 子模块设计完成后补全，并与工作台「项目状态」共用同一取值。

### 2.6 步骤页签联动

页签按采购方式 × 公开性联动（对照 [README 2.1 全流程对照矩阵](README.md)），各场景步骤页签序列如下：

| 采购方式 × 公开性 | 步骤页签序列 |
| --- | --- |
| 招标-公开 | 招标公告 → 招标文件 → 标前准备 → 开标 → 评标 → 定标 → 标后 |
| 招标-邀请 | 邀请函 → 招标文件 → 标前准备 → 开标 → 评标 → 定标 → 标后 |
| 询比-公开 | 询比公告 → 采购文件 → 标前准备 → 开启 → 评审 → 成交 → 成交后 |
| 询比-邀请 | 邀请函 → 采购文件 → 标前准备 → 开启 → 评审 → 成交 → 成交后 |
| 谈判-公开 | 谈判公告 → 采购文件 → 标前准备 → 开启 → 评审 → 成交 → 成交后 |
| 谈判-邀请 | 邀请函 → 采购文件 → 标前准备 → 开启 → 评审 → 成交 → 成交后 |
| 竞价-公开 | 竞价公告 → 竞价准备 → 竞价 → 成交 → 成交后 |
| 竞价-邀请 | 竞价邀请 → 竞价准备 → 竞价 → 成交 → 成交后 |
| 直接采购 | 事前公示 → 邀请函 → 采购文件 → 标前准备 → 开启 → 评审 → 成交 → 成交后 |

- 竞价：无采购文件、无评审，公告后直接竞价准备 → 竞价 → 成交；
- 直接采购：含「事前公示」「邀请函」两个独立页签。

### 2.7 页签规则

- 纯文字页签，不标进度状态；
- 未到达的页签**禁用置灰**，不可点击；
- 当前所在页签高亮。
