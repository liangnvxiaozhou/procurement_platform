# 平台使用费收费标准配置

> 设计日期：2026-06-04
> 最后修改：2026-06-04 16:30:00
---

## 一、功能概述

平台使用费收费标准配置为每个租户提供多套阶梯式收费规则管理能力。采用"项目类别（招标/非招）× 组织形式（自行采购/委托采购）"四维矩阵，支持每个维度下维护多条规则历史，同一时间仅 1 条规则生效。

功能入口：租户列表操作列 → 平台使用费 → 打开独立页面，完成四类收费规则的阶梯配置及生效规则管理。

```
租户管理 → 平台使用费
          │ 打开独立页面
          ▼
┌─ 平台收费标准 ──────────────────────────────────────────────┐
│ [自采招标] [自采非招] [委托招标] [委托非招]                 │
│   规则列表（多条规则，1 条生效）                              │
│   支持：新增 / 编辑 / 生效 / 失效 / 删除                    │
└─────────────────────────────────────────────────────────────┘
          │ 读取生效规则
          ▼
采购立项 → 标段包（根据项目类别+组织形式定位当前生效规则并计费）
```

## 二、数据结构

### 2.1 平台使用费规则主表（platform_fee_rule）

| 字段   | 类型   | 必填    | 说明                     |
| ---- | ---- | ----- | ---------------------- |
| 规则名称 | 文本   | ✅     | 用户手填                   |
| 规则编号 | 文本   | ✅系统自动 | 前缀+序号，按组织形式区分（见下表）     |
| 计算方式 | 下拉选择 | ✅系统自动 | 当前固定为"梯度进制"，预留后续扩展；只读  |
| 计算金额 | 下拉选择 | ✅     | 预算价 / 成本价              |
| 项目类别 | 下拉选择 | ✅系统自动 | 招标 / 非招（Tab 锁定，只读）     |
| 组织形式 | 下拉选择 | ✅系统自动 | 自行采购 / 委托采购（Tab 锁定，只读） |
| 是否计费 | 单选   | ✅     | 收取 / 不收取，默认"收取"        |
| 是否生效 | 单选   | ✅     | 生效 / 无效，默认"无效"         |
| 租户ID | 外键   | ✅系统自动 | 所属租户                   |
| 创建人  | 文本   | ✅系统自动 | 创建用户姓名                 |
| 创建时间 | 日期时间 | ✅系统自动 | yyyy-mm-dd HH:mm:ss    |
| 修改人  | 文本   | ✅系统自动 | 最后更新用户姓名               |
| 修改时间 | 日期时间 | ✅系统自动 | yyyy-mm-dd HH:mm:ss    |

**规则编号前缀表**：

| Tab | 前缀示例 |
|-----|---------|
| 自采招标 | `FEE-ZB-ZX-001` |
| 委托招标 | `FEE-ZB-WT-001` |
| 自采非招 | `FEE-FZB-ZX-001` |
| 委托非招 | `FEE-FZB-WT-001` |

### 2.2 阶梯规则表（platform_fee_tier）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 规则ID | 外键 | ✅ | 关联主表 |
| 序号 | 整数 | ✅系统自动 | 按最小值升序排序 |
| 最小值（万元） | 数字 | ✅ | 左闭，含本值 |
| 最大值（万元） | 数字 / 特殊值 | ✅ | 右开，不含本值；最后一档可选"无上限" |
| 收费金额（元） | 数字 | ✅ | 固定金额，精确到分 |

#### 阶梯示例

| 序号 | 最小值（万元） | 最大值（万元） | 收费金额（元） |
|------|---------------|---------------|---------------|
| 1 | 0 | 100 | 2,000 |
| 2 | 100 | 500 | 3,500 |
| 3 | 500 | 1,000 | 5,000 |
| 4 | 1,000 | 无上限 | 8,000 |

**计费计算示例**：某招标项目预算价 350 万元，匹配第 2 档 [100, 500)，平台使用费为 3,500 元。

**Tab 默认行为**：进入页面默认显示第一个 Tab（自采招标），如某 Tab 无规则则显示空状态提示。

**操作列逻辑**：

| 操作 | 条件 | 说明 |
|------|------|------|
| 编辑 | 始终可用 | 打开规则详情抽屉 |
| 删除 | 是否生效 = 无效 | 已生效的规则不可删除（按钮置灰） |
| 生效 | 是否生效 = 无效 | 点击后同 Tab 其他规则自动失效，当前规则生效 |
| 失效 | 是否生效 = 生效 | 点击后当前规则变为无效（该 Tab 下无生效规则） |

**"是否计费"联动逻辑**：
- 选择"收取"：计算规则区域展开，可配置阶梯（计费方式按"计算方式"字段执行，当前固定梯度进制）
- 选择"不收取"：计算规则区域折叠隐藏，已配置的阶梯数据清空

### 3.4 添加/编辑档位弹窗

```
┌─ 添加档位 ───────────────────────────────────────┐
│                                                  │
│  * 最小值（万元）  [                  ]          │
│  * 最大值（万元）  [                  ]          │
│    ☐ 无上限（勾选后最大值不可编辑）               │
│  * 收费金额（元）  [                  ]          │
│                                                  │
│                [ 取消 ]  [ 确定 ]                │
└──────────────────────────────────────────────────┘
```

**说明**：
- 第一个档位的最小值固定为 0，只读
- 后续档位的最小值自动带入上一档的最大值，只读
- 勾选"无上限"后最大值输入框置灰，且只允许最后一个档位勾选
- 每个规则至少需要一个档位

## 四、业务规则

### 4.1 阶梯校验规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R1 | 最小值起始 | 第一个档位的最小值固定为 0 |
| R2 | 阶梯连续性 | 后续档位的最小值自动带入上一档的最大值 |
| R3 | 区间有效 | 最大值必须大于最小值 |
| R4 | 无上限唯一 | "无上限"只能出现在最后一个档位，且整个规则只能有一个 |
| R5 | 金额正数 | 收费金额必须为正数，精确到分 |
| R6 | 条件阶梯 | 仅当"是否计费"= 收取 时，规则必须至少包含一个阶梯档位 |
| R7 | 收费递增提示 | 当收费金额未随区间增大而递增时，弹出确认提示（不强制） |
| R8 | 编号唯一 | 规则编号全局唯一 |

### 4.2 规则生命周期规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R9 | 建议配置 | 建议每个租户为四个 Tab（项目类别×组织形式）各配置至少一条规则，以确保计费逻辑完整 |
| R10 | 唯一生效 | 每个 Tab（项目类别×组织形式）下最多只能有一条规则处于"生效"状态 |
| R11 | Tab 独立 | 四个 Tab 的规则生效状态相互独立，互不影响 |
| R12 | 切换即时生效 | 租户修改收费规则后，新立项项目按新规则计费；已立项项目不受影响（已快照当时规则） |

### 4.3 计费判断规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R13 | 计费逻辑 | 项目立项时，根据项目类别和组织形式定位对应 Tab 下的生效规则；"是否计费" = 收取 时，按计费方式（当前梯度进制）计算平台使用费 |
| R14 | 不收取处理 | 若生效规则的"是否计费"字段为"不收取"，则平台使用费为 0 元，不进行阶梯计算 |
| R15 | 代理服务费独立 | 代理服务费与平台使用费独立，代理服务费在标段信息中配置，不在本规则中处理 |

### 4.4 示例场景：集团 A

**计费维度**：项目类别（招标/非招）× 组织形式（自行采购/委托采购）= 4 种组合

#### 收费规则配置

- **招标类规则**：[0, 无上限) → 0 元（实质免费），"是否计费" = 收取，"是否生效" = 生效
- **非招类规则**：(0, 5] → 0 元；(5, 20] → 800 元；(20, +∞) → 1,200 元，"是否计费" = 收取，"是否生效" = 生效
- **委托招标规则**："是否计费" = 不收取，"是否生效" = 生效
- **委托非招规则**："是否计费" = 不收取，"是否生效" = 生效

#### 计费矩阵

| 项目类别 | 组织形式 | 平台使用费 | 说明 |
|---------|---------|-----------|------|
| 招标 | 自行采购 | 0 元 | 按招标规则，全档位 0 元 |
| 非招 | 自行采购 | 按阶梯收费 | 按非招规则：(0,5]→0；(5,20]→800；(20,∞)→1200 |
| 招标 | 委托采购（代理） | 0 元 | "是否计费" = 不收取，不收平台使用费 |
| 非招 | 委托采购（代理） | 0 元 | "是否计费" = 不收取，不收平台使用费 |

**补充说明**：
- 委托采购（代理）项目的代理服务费在标段信息中配置（当前需求文档中尚未涉及委托场景，待后续设计）
- 每个 Tab 下的规则生效状态独立管理，可灵活配置不同场景的收费策略

## 五、与其他功能的关系

### 5.1 项目立项

- 采购立项 → 标段包中的"平台使用费收费规则"字段，自动读取当前租户对应 Tab（项目类别×组织形式）下的生效规则
- 若生效规则的"是否计费"字段为"收取"，则按计费方式（当前固定梯度进制）计算平台使用费
- 若生效规则的"是否计费"字段为"不收取"，则平台使用费为 0 元
- 字段为只读，仅展示规则名称和计费方式

### 5.2 全局日志审计

- 规则的新建、编辑、删除、生效/失效操作记录在控制中心的全局操作日志中
- 记录操作人、操作时间、操作类型、规则编号、规则状态变更
