# 平台使用费收费标准配置

> 设计日期：2026-06-04
> 最后修改：2026-06-04 15:00:00
---

## 一、功能概述

平台使用费收费标准配置用于为每个租户创建阶梯式收费规则。每个租户需要配置一套招标类规则和一套非招标类规则，作为该租户所有采购项目的平台使用费计费依据。

功能入口位于租户列表操作列的"平台使用费"按钮，点击后打开配置抽屉，完成两类规则的阶梯配置及代理项目收费开关设置。

```
租户管理 → 平台使用费（配置招标类+非招标类阶梯规则）
                    │
                    │ 写入租户收费规则（含代理项目收费开关）
                    ▼
采购立项 → 标段包（读取收费规则，根据组织形式判断是否计费）
```

## 二、数据结构

### 2.1 平台使用费规则主表（platform_fee_rule）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 规则名称 | 文本 | ✅ | 用户手填 |
| 规则编号 | 文本 | ✅系统自动 | 前缀+序号，招标类 `FEE-ZB-XXX`，非招类 `FEE-FZB-XXX` |
| 计算方式 | 文本 | ✅系统自动 | 固定为"梯度进制"，只读 |
| 计算金额 | 下拉选择 | ✅ | 计费依据：预算价 / 成本价 |
| 项目类别 | 下拉选择 | ✅ | 招标 / 非招 |
| 租户ID | 外键 | ✅系统自动 | 所属租户 |
| 创建人 | 文本 | ✅系统自动 | 创建用户姓名 |
| 创建时间 | 日期时间 | ✅系统自动 | yyyy-mm-dd HH:mm:ss |
| 修改人 | 文本 | ✅系统自动 | 最后更新用户姓名 |
| 修改时间 | 日期时间 | ✅系统自动 | yyyy-mm-dd HH:mm:ss |

### 2.2 阶梯规则表（platform_fee_tier）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 规则ID | 外键 | ✅ | 关联主表 |
| 序号 | 整数 | ✅系统自动 | 按最小值升序排序 |
| 最小值（万元） | 数字 | ✅ | 左闭，含本值 |
| 最大值（万元） | 数字 / 特殊值 | ✅ | 右开，不含本值；最后一档可选"无上限" |
| 收费金额（元） | 数字 | ✅ | 固定金额，精确到分 |

#### 阶梯示例

| 序号 | 最小值（万元） | 最大值（万元） | 收费金额（元） |
|------|---------------|---------------|---------------|
| 1 | 0 | 100 | 2,000 |
| 2 | 100 | 500 | 3,500 |
| 3 | 500 | 1,000 | 5,000 |
| 4 | 1,000 | 无上限 | 8,000 |

**计费计算示例**：某招标项目预算价 350 万元，匹配第 2 档 [100, 500)，平台使用费为 3,500 元。

### 2.3 租户收费配置表（tenant_fee_config）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 租户ID | 外键 | ✅系统自动 | 所属租户 |
| 代理项目收取平台使用费 | 布尔 | ✅ | 开/关，默认关 |

**说明**：
- 每个租户一条记录，首次打开平台使用费配置抽屉时自动创建
- 预留扩展能力，后续按需增加更多开关字段

### 3.3 添加/编辑档位弹窗

```
┌─ 添加档位 ───────────────────────────────────────┐
│                                                  │
│  * 最小值（万元）  [                  ]          │
│  * 最大值（万元）  [                  ]          │
│    ☐ 无上限（勾选后最大值不可编辑）               │
│  * 收费金额（元）  [                  ]          │
│                                                  │
│                [ 取消 ]  [ 确定 ]                │
└──────────────────────────────────────────────────┘
```

**说明**：
- 第一个档位的最小值固定为 0，只读
- 后续档位的最小值自动带入上一档的最大值，只读
- 勾选"无上限"后最大值输入框置灰，且只允许最后一个档位勾选
- 每个规则至少需要一个档位

## 四、业务规则

### 4.1 阶梯校验规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R1 | 最小值起始 | 第一个档位的最小值固定为 0 |
| R2 | 阶梯连续性 | 后续档位的最小值自动带入上一档的最大值 |
| R3 | 区间有效 | 最大值必须大于最小值 |
| R4 | 无上限唯一 | "无上限"只能出现在最后一个档位，且整个规则只能有一个 |
| R5 | 金额正数 | 收费金额必须为正数，精确到分 |
| R6 | 至少一档 | 每套规则必须至少包含一个阶梯档位 |
| R7 | 收费递增提示 | 当收费金额未随区间增大而递增时，弹出确认提示（不强制） |
| R8 | 编号唯一 | 规则编号全局唯一 |
| R9 | 租户必填 | 每个租户必须配置招标类和非招标类两套规则 |
| R10 | 切换即时生效 | 租户修改收费规则后，新立项项目按新规则计费；已立项项目不受影响（已快照当时规则） |

### 4.2 代理项目计费规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R11 | 代理项目收费逻辑 | "代理项目收取平台使用费"开关关闭时，委托采购（代理）项目不收平台使用费（0元）；开关开启时，复用招标/非招阶梯规则 |
| R12 | 计费判断顺序 | 项目立项时，先判断项目类别（招标/非招）选择对应规则，再判断组织形式（自行采购/委托采购），若为委托采购且开关关闭则计费为0 |
| R13 | 代理服务费独立| 代理服务费与平台使用费独立，代理服务费在标段信息中配置，不在本规则中处理 |

### 4.3 示例场景：集团A

**计费维度**：项目类别（招标/非招）× 组织形式（自行采购/委托采购）= 4种组合

#### 收费规则配置

- **招标类规则**：[0, 无上限) → 0 元（实质免费）
- **非招类规则**：(0, 5] → 0元；(5, 20] → 800元；(20, +∞) → 1,200元
- **代理项目收取平台使用费**：**关**

#### 计费矩阵

| 项目类别 | 组织形式     | 平台使用费 | 说明                                   |
| ---- | -------- | ----- | ------------------------------------ |
| 招标   | 自行采购     | 0 元   | 按招标规则，全档位0元                          |
| 非招   | 自行采购     | 按阶梯收费 | 按非招规则：(0,5]→0；(5,20]→800；(20,∞)→1200 |
| 招标   | 委托采购（代理） | 0 元   | 开关关闭，不收平台使用费                         |
| 非招   | 委托采购（代理） | 0 元   | 开关关闭，不收平台使用费                         |

**补充说明**：
- 委托采购（代理）项目的代理服务费在标段信息中配置（当前需求文档中尚未涉及委托场景，待后续设计）
- 如果开关开启，委托采购将与自行采购共用同一套招标/非招阶梯规则

## 五、与其他功能的关系

### 5.1 项目立项

- 采购立项 → 标段包中的"平台使用费收费规则"字段，自动读取当前租户的收费规则
- 根据项目的采购方式判断显示招标类或非招标类规则（参见《项目立项功能设计》3.2 标段包）
- 字段为只读，仅展示规则名称

### 5.2 全局日志审计

- 规则的新建、编辑操作记录在控制中心的全局操作日志中
