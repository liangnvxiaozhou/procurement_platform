## 一、功能概述

### 1.1 业务定位

项目立项是采购流程中承上启下的关键环节。采购立项审核通过后，项目经理在「我的项目」页面基于已生效的采购立项信息建立正式采购项目，并划分标段包、关联项目明细，为后续招标采购执行提供项目基础数据。

### 1.2 与采购立项的关系

| 维度 | 采购立项 | 项目立项（本模块） |
|------|---------|------------------|
| 前置条件 | 可执行采购计划（子计划/项目计划） | 已生效的采购立项 |
| 操作角色 | 立项提报人 | 项目经理 |
| 核心动作 | 选择计划及明细 → 填写立项信息 → 提交审核 | 选择采购立项 → 带出立项信息 → 添加标段包 → 添加明细 → 确认 |
| 是否需要审核 | 需要（本级/二级审批，必要时采购服务中心复核） | 不需要（保存后直接确认即可） |
| 状态流转 | 草稿 → 待审核 → 已生效/已复核 | 草稿 → 已确认 |
| 后续环节 | 被项目立项关联使用 → 已建项 | 进入招标采购执行环节 |

### 1.3 模块划分

```
项目立项管理
│
└── 我的项目
    ├── 项目列表（查看、搜索、筛选）
    ├── 新建项目（选择采购立项 → 填写标段包 → 添加明细）
    ├── 项目详情（查看项目信息、标段包、明细）
    └── 撤销（已确认项目可撤销回草稿）
```

## 二、核心业务流程

### 2.1 项目立项流程

```
项目经理进入我的项目 → 点击新建 → 弹窗选择已生效采购立项 → 
系统自动带出立项信息 → 添加标段包 → 填写标段包信息 → 
标段包下添加项目明细 → 保存（草稿）→ 确认（已确认）→ 进入项目执行
```

### 2.2 状态流转图

```
首次创建：
  新建 → 填写信息 → 保存 → 草稿 → 确认 → 已确认 → 进入项目执行

撤销流程：
  已确认 → 撤销 → 草稿 → 编辑 → 保存 → 草稿 → 确认 → 已确认

删除流程：
  草稿 → 删除 → 数据移除
```

## 六、关键业务规则

### 6.1 建项规则

| 规则 | 说明 |
|------|------|
| 前置条件 | 仅可选择状态为「已生效」或「已复核」的采购立项 |
| 立项复用 | 一个采购立项可建项目数量待确认 |
| 信息带出 | 选择立项后自动带出立项信息，部分字段可编辑 |
| 标段包约束 | 至少添加一个标段包方可确认 |
| 明细约束 | 标段包下至少添加一条明细方可保存标段包 |
| 明细来源 | 项目明细从关联的采购立项明细中选择，不可手动新增 |
| 明细复用 | 同一条采购立项明细可被多个标段包使用？待确认 |

### 6.2 保存与确认规则

| 操作 | 行为 | 状态变化 |
|------|------|---------|
| 保存 | 保存当前填写内容，校验非必填项 | 新建 → 草稿 |
| 确认 | 校验所有必填项，通过后完成建项 | 草稿 → 已确认 |

**确认时校验项**：
- 项目名称已填写
- 至少包含一个标段包
- 每个标段包下至少包含一条明细
- 标段包必填项均已填写

### 6.3 撤销规则

| 条件 | 行为 |
|------|------|
| 已确认状态 | 可撤销回草稿状态 |
| 撤销后 | 状态变为草稿，可继续编辑 |
| 不可撤销条件 | 待确认（如已发布公告/已开标等） |

### 6.4 编号生成规则

| 编号类型 | 规则 |
|---------|------|
| 项目编号 | 系统自动生成，规则待定 |
| 标段包编号 | 系统自动生成，规则待定 |

## 七、状态字典

### 7.1 项目状态

| 状态 | 操作 | 说明 |
|------|------|------|
| 草稿 | 编辑 / 删除 | 编制中，未完成建项 |
| 已确认 | 查看 / 撤销 / 进入项目 | 建项完成，可进入后续采购执行环节 |

### 7.2 状态流转图

```
首次创建：
  新建 → 填写信息 → 保存 → 草稿 → 确认 → 已确认

撤销流程：
  已确认 → 撤销 → 草稿 → 编辑 → 保存 → 草稿 → 确认 → 已确认

删除流程：
  草稿 → 删除 → 数据移除
```
