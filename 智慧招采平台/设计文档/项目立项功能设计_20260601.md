## 一、功能概述

采购立项审核通过后，项目经理在「我的项目」页面基于已生效的采购立项建立正式采购项目，划分标段包、关联项目明细，为后续招标采购执行提供基础数据。

```
项目立项管理
│
└── 我的项目
    ├── 项目列表（查看、搜索、筛选）
    ├── 新建项目（选择采购立项 → 填写标段包 → 添加明细）
    ├── 项目详情（查看项目信息、标段包、明细）
    └── 撤销（已确认项目可撤销回草稿）
```

---

## 二、核心业务流程

```
点击新建 → 弹窗选择已生效采购立项 → 进入项目详情页（编辑模式）→ 
编辑项目名称 → 保存（草稿）→ 返回列表 → 
点击编辑详情 → 完善项目信息 → 添加标段包 → 填写标段包信息 → 
标段包下添加项目明细 → 保存 → 确认（已确认）→ 进入项目执行
```

**区域1：关联采购立项**

展示关联的采购立项名称及编号，只读不可修改。

---

**区域2：项目信息**

字段定义参见 3.1。选择立项后自动带出。

可编辑字段：项目名称、所在行政区域、项目实施地点、项目性质、询比方式、采购依据、项目概况、其他、附件。
只读字段：项目类型、采购方式、本项目行业分类、组织形式、项目经理。

**采购方式联动字段**：
- 采购方式=询比采购 → 显示询比方式（公开/邀请），可编辑
- 资审方式、邀请供应商、谈判方式、直接采购属性、竞价方式本期暂不启用

## 六、关键业务规则

### 6.1 建项规则

- 仅可选择状态为「已生效」或「已复核」的采购立项
- 一个采购立项可建项目数量待确认
- 选择立项后进入项目详情页（编辑模式），编辑项目名称即可保存为草稿
- 草稿项目通过「编辑详情」继续完善：项目信息、标段包、明细
- 至少添加一个标段包方可确认
- 每个标段包下至少添加一条明细方可保存标段包
- 项目明细从关联的采购立项明细中选择，不可手动新增
- 同一条采购立项明细是否可被多个标段包使用，待确认
- 详情页中关联的采购立项名称不可修改

### 6.2 保存与确认

| 操作 | 行为 | 状态变化 |
|------|------|---------|
| 保存 | 保存当前内容 | → 草稿 |
| 确认 | 校验必填项：项目名称、所在行政区域、项目实施地点、项目性质、项目概况、其他、询比方式（采购方式=询比采购时）、至少一个标段包、每个标段包至少一条明细、标段包必填项 | → 已确认 |

### 6.3 撤销

已确认状态可撤销回草稿，撤销后可继续编辑。不可撤销的条件待确认（如已发布公告/已开标等）。

### 6.4 编号生成

项目编号、标段包编号均由系统自动生成，规则待定。
