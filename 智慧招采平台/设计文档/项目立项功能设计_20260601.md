## 一、功能概述

采购立项审核通过后，项目经理在「我的项目」页面基于已生效的采购立项建立正式采购项目，划分标段包、关联项目明细，为后续招标采购执行提供基础数据。

```
采购系统
│
├── 采购立项（略，详见采购立项功能设计文档）
│
└── 项目管理
    └── 我的项目
        ├── 项目列表（查看、搜索、筛选）
        ├── 新建项目（选择采购立项 → 自动创建项目）
        └── 项目详情（完善信息、添加标段包、添加明细、确认）
```

---

## 二、核心业务流程

```
点击新建 → 选择采购立项 → 自动创建项目（草稿）→ 进入项目详情页（编辑模式）

在详情页中（支持分段保存）：
  完善项目信息 → 添加标段包（轻量弹窗）→ 添加明细 → 
  完善标段包信息

点击确认 → 完整度校验 → 状态变为已确认 → 进入项目执行
```

**评审办法业务含义**：
- **经评审的最低投标价法**：投标先通过全部实质性审查（工期、质量、资质、技术标准全部达标），再把报价做合规微调（缺项、商务偏差加价），经评审总价最低者中标，低于自身成本报价直接作废。适用于通用标准化项目：普通土建劳务、标准建材、常规设备、简单修缮、管线铺装，技术统一、无特殊定制要求。
- **综合评估法**：总分100分，分报价分、技术分、商务分三板块加权计分，总分第一名中标，价格只是评分其中一项。适用于技术复杂、定制化项目：大型EPC总承包、智能化、机电成套设备、设计类、精装修、特种工程、软件开发。
- **经评审的最高投标价法**：国标法定招标（工程、政府采购公开招标）没有这个法定评标办法，只用于资产出让、出租类竞拍，不用于工程施工、货物采购招标。本期不启用。

**报价方式业务含义**：
- **金额报价（总价包干/固定总价）**：直接报总金额，一口价。
- **费率报价（百分比报价）**：不报多少钱，报费率%，最终结算=审定造价×中标费率。例：造价咨询报费率1.2‰、监理报费率2.3%。
- **单价报价（清单单价，按实际量结算）**：报单个多少钱，最终总价=实际发生工程量×中标单价。

**联合体投标业务含义**：
- 两家及以上独立公司签联合协议，捆绑成一个投标主体投同一个项目，中标后一起履约、连带担责。
- 签了联合体协议后，所有成员不能再单独投标、也不能再加入别的联合体投本项目，违规全部废标。
- 标书里必须附上协议，写明：牵头单位、各方分工、权责、付款、风险划分，缺协议直接废标。
- 中标后全体一起签合同，甲方找任何一家追责都有效：一方违约/烂尾，其余联合体成员要兜底赔偿。

### 3.3 项目明细

从采购立项明细中选择后自动带出，只读不可编辑。标段包与明细是一对多关系，一条明细只能关联一个标段包，被使用后不可再被其他标段包使用。

**工程/服务类**：项目名称、项目编号、项目范围、工程量、估算金额（万元）、计划采购时间、开工/交货日期

**物资类**：物料名称、物料编号、规格型号、计量单位、采购数量、预估单价（元）、估算金额（万元）、计划采购时间、到货时间

## 六、关键业务规则

### 6.1 建项规则

- 仅可选择状态为「已生效」或「已复核」的采购立项
- 选择采购立项后自动创建项目（草稿状态），项目名称自动带出，并进入详情页（编辑模式）
- 支持分段保存：用户可分多次完善项目信息、标段包、明细，每次保存仅保存当前内容
- 标段包创建时仅需填写标段包名称，其余字段在详情面板中完善
- 点击「确认」时进行完整度校验，校验通过后状态变为已确认
- 至少添加一个标段包方可确认
- 每个标段包下至少添加一条明细方可确认
- 项目明细从关联的采购立项明细中选择，不可手动新增
- 标段包与明细是一对多关系，一条明细只能关联一个标段包，被使用后不可再被其他标段包使用
- 详情页中关联的采购立项名称不可修改
- 删除项目后，已关联的采购立项将被释放，可重新用于建项
- 项目组成员可在「我的项目」列表看到该项目，操作权限等同项目经理
- 项目经理字段受保护，不可被任何成员删除

### 6.2 保存与确认

- **分段保存**：编辑页面任何时刻均可保存当前内容，状态保持草稿，用户可分多次完善。
- **完整度校验**：点击「确认」时一次性校验以下必填项，通过后状态变为已确认：
  - 项目名称、项目概况已填写
  - 询比方式已填写（采购方式=询比采购时）
  - 至少一个标段包
  - 每个标段包必填项均已填写（标段包名称、采购流程控制、评审办法、标书使用编制工具、满足开启家数、是否报名、报价方式、文件是否出售、是否缴纳保证金、联合体投标、工期、工期说明、招标采购范围、供应商基本要求、供应商资质要求、供应商业绩要求、供应商其他要求）
  - 每个标段包下至少有一条明细

### 6.3 撤销

已确认状态可撤销回草稿，撤销后可继续编辑。不可撤销的条件待确认（如已发布公告/已开标等）。

### 6.4 删除

草稿状态可删除项目。删除前弹出二次确认弹窗：「确认删除该项目？删除后不可恢复，已关联的采购立项将被释放，可重新用于建项。」确认后删除项目及其所有标段包和明细数据。

### 6.5 编号生成

项目编号、标段包编号均由系统自动生成，规则待定。标段包编号在添加标段包时立即生成。

| 草稿 | 编辑 / 成员 / 删除 | 编制中 |

