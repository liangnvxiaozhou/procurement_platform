## 一、功能概述

采购立项审核通过后，项目经理在「我的项目」页面基于已生效的采购立项建立正式采购项目，划分标段包、关联项目明细，为后续招标采购执行提供基础数据。

```
项目立项管理
│
└── 我的项目
    ├── 项目列表（查看、搜索、筛选）
    ├── 新建项目（选择采购立项 → 填写标段包 → 添加明细）
    ├── 项目详情（查看项目信息、标段包、明细）
    └── 撤销（已确认项目可撤销回草稿）
```

---

## 二、核心业务流程

```
点击新建 → 弹窗选择已生效采购立项 → 系统自动带出立项信息 → 
添加标段包 → 填写标段包信息 → 标段包下添加项目明细 → 
保存（草稿）→ 确认（已确认）→ 进入项目执行
```

## 六、关键业务规则

### 6.1 建项规则

- 仅可选择状态为「已生效」或「已复核」的采购立项
- 一个采购立项可建项目数量待确认
- 至少添加一个标段包方可确认
- 每个标段包下至少添加一条明细方可保存
- 项目明细从关联的采购立项明细中选择，不可手动新增
- 同一条采购立项明细是否可被多个标段包使用，待确认

### 6.2 保存与确认

| 操作 | 行为 | 状态变化 |
|------|------|---------|
| 保存 | 保存当前内容 | → 草稿 |
| 确认 | 校验必填项：项目名称、至少一个标段包、每个标段包至少一条明细、标段包必填项 | → 已确认 |

### 6.3 撤销

已确认状态可撤销回草稿，撤销后可继续编辑。不可撤销的条件待确认（如已发布公告/已开标等）。

### 6.4 编号生成

项目编号、标段包编号均由系统自动生成，规则待定。
