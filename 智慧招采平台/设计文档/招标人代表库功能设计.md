# 招标人代表库功能设计

> 设计日期：2026-06-02
> 最后修改：2026-06-02

---

## 一、功能概述

招标人代表库用于企业管理员集中管理本企业的招标人代表信息，支持将招标人代表共享给其他企业使用，也可查看其他企业共享给本企业的招标人代表。

```
平台管理中心 → 企业管理 → 招标人代表库
│
├── 1. 本企业创建
│   ├── 列表查看（搜索、筛选）
│   ├── 添加招标人代表（从本单位启用用户中选择）
│   ├── 编辑招标人代表
│   ├── 查看招标人代表详情
│   ├── 删除招标人代表
│   └── 共享招标人代表（给其他企业）
│
└── 2. 其他企业共享
    ├── 列表查看（搜索、筛选）
    └── 查看招标人代表详情（仅查看和使用，不可编辑）
```

---

## 二、核心业务流程

### 2.1 添加流程

```
列表页点击【添加】→ 打开添加弹窗 → 选择姓名（搜索本单位启用用户）→
自动回显身份证号码、所属单位、联系电话 → 编辑所属地区（省/市/区）→
设置是否有效（开关）→ 填写银行账户等可选信息 → 点击【确定】→
系统自动生成招标人代表编号 → 返回列表页
```

### 2.2 编辑流程

```
列表页点击【编辑】→ 打开编辑弹窗 → 回显已保存数据 →
修改所属地区、是否有效、开户行、银行账户、银行联行号、备注 →
点击【确定】→ 系统自动记录修改人和修改时间 → 返回列表页
```

### 2.3 共享流程

```
列表页点击【共享】→ 打开共享弹窗 → 回显已共享企业列表 →
点击【添加】→ 打开企业选择弹窗 → 按企业名称/企业代码搜索 →
勾选目标企业 → 点击【确定】→ 返回共享弹窗 →
点击【关闭】→ 返回列表页

取消共享：
共享弹窗中点击【删除】→ 二次确认 → 取消该企业的共享关系
```

### 2.4 接收共享流程

```
其他企业共享招标人代表给本企业 → 本企业切换到"其他企业共享"Tab →
查看共享的招标人代表信息 → 仅可查看和使用，不可编辑或删除
```

## 三、数据结构

### 3.1 招标人代表主表（bid_representative）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 招标人代表编号 | 文本 | - | 系统自动生成，创建后回显，唯一标识 |
| 姓名 | 下拉选择 | ✅ | 数据源：sys_user 中本企业状态为启用的用户，支持搜索 |
| 身份证号码 | 文本 | - | 选择姓名后自动回显，只读 |
| 所属单位 | 文本 | - | 选择姓名后自动回显，只读 |
| 联系电话 | 文本 | - | 选择姓名后自动回显，只读 |
| 所属地区 | 级联选择 | ✅ | 省/市/区，三级联选 |
| 是否有效 | 开关 | ✅ | 有效/无效，默认有效 |
| 开户行 | 文本 | ❌ | |
| 银行账户 | 文本 | ❌ | |
| 银行联行号 | 文本 | ❌ | |
| 备注 | 文本域 | ❌ | |
| 创建人 | 文本 | 系统自动 | 创建用户姓名 |
| 创建时间 | 日期时间 | 系统自动 | yyyy-mm-dd HH:mm:ss |
| 修改人 | 文本 | 系统自动 | 最后修改用户姓名，每次保存自动更新 |
| 修改时间 | 日期时间 | 系统自动 | yyyy-mm-dd HH:mm:ss，每次保存自动更新 |
| 所属组织 | 文本 | 系统自动 | 创建时自动填充本企业 org_id |

**说明**：
- 姓名、身份证号码、所属单位、联系电话四个字段来源于 sys_user 表，创建后姓名不可修改
- 身份证号码完整展示，不做脱敏处理
- 修改人和修改时间仅在查看页面展示，编辑时不展示
- 每次编辑保存时，系统自动将当前操作人写入修改人字段，并更新修改时间

### 3.2 共享关系表（bid_representative_share）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | 文本 | 系统自动 | 主键 |
| bid_representative_id | 文本 | ✅ | 关联招标人代表主表 |
| target_org_id | 文本 | ✅ | 被共享目标企业ID |
| target_org_name | 文本 | ✅ | 被共享目标企业名称 |
| target_org_code | 文本 | ✅ | 被共享目标企业代码（社会信用代码） |
| 创建人 | 文本 | 系统自动 | 创建共享关系时记录 |
| 创建时间 | 日期时间 | 系统自动 | yyyy-mm-dd HH:mm:ss |

**说明**：
- 同一个招标人代表不可重复共享给同一家企业
- 删除共享记录后，接收方企业的"其他企业共享"列表中不再显示该招标人代表
- 共享关系仅为单向共享，接收方不可再次共享给第三方企业

**保存规则**：

- 保存成功后关闭弹窗，列表刷新并显示新增的记录

### 5.5 共享弹窗

**打开方式**：列表页"本企业创建"Tab 点击操作列【共享】按钮

- 被共享企业仅可查看和使用该招标人代表的信息，不可编辑
- 被共享企业不可将接收到的招标人代表再次共享给第三方企业
- 本企业将招标人代表设为无效后，已共享企业的接收记录同步失效（状态同步为无效）

## 六、待确认事项

| 序号 | 待确认内容 | 建议方案 |
| --- | --- | --- |
| 1 | 删除招标人代表时，是否需要校验该代表是否已被招标项目关联使用（如已使用则提示"该招标人代表已被项目使用，无法删除"，还是直接删除） | 建议校验，已被使用时拦截删除 |
| 2 | 招标人代表编号的具体生成规则（如"ZBD"+日期+自增序号：ZBD202606020001，还是按年度自增、按组织自增） | 建议采用"ZBD"+年+月+4位自增序号 |
| 3 | 被取消共享的招标人代表，在接收方企业侧是否保留历史记录还是直接消失 | 建议直接消失，不保留历史记录 |
| 4 | 本企业将招标人代表设为无效后，对已共享企业的接收记录是同步失效还是保持独立状态 | 建议同步失效，状态跟随创建方 |
| 5 | 添加时是否需要校验同一个用户不能重复添加到招标人代表库 | 建议校验，同一企业下同一用户不可重复 |
