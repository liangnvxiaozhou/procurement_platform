## 一、功能概述

采购立项模块用于基于已生效的可执行采购计划，拆分到明细级别创建具体的采购项目，是采购计划到采购执行的衔接环节。

本模块支持基于可执行计划中的子计划/项目计划进行采购项目立项，基于可执行电煤计划中的市场煤计划进行市场煤采购立项，以及基于可执行电煤计划中的长协煤计划进行长协煤采购创建（协议订单管理）。

### 模块划分

```
采购立项管理
│
├── 1. 采购项目立项
│   └── 基于可执行计划中的子计划/项目计划，选择明细进行立项
│
├── 2. 市场煤采购立项
│   └── 基于可执行电煤计划中的市场煤计划，选择电煤明细进行立项
│
└── 3. 长协煤采购创建（协议订单管理）
    └── 基于可执行电煤计划中的长协煤计划，创建协议订单
    （注：长协煤为协议性合作，不需要招投标）

审核入口：采购系统 → 任务中心（统一处理所有立项审核）
```

---

## 二、核心业务流程

### 2.1 采购项目立项流程

```
选择计划来源（年度采购子计划/项目计划）→ 选择计划 → 选择明细 → 填写立项信息 → 提交审核 → 生效
```

**约束**：
- 一条明细仅用一次
- 所选明细的项目类型须一致
- 拟采购方式自动带入，可编辑

### 2.2 市场煤采购立项流程

```
选择市场煤计划 → 选择电煤明细 → 填写市场煤采购立项信息 → 提交审核 → 生效
```

**约束**：
- 一条电煤明细仅用一次
- 目前仅支持市场煤立项

**立项状态**：

| 状态    | 操作         | 说明                             |
| ----- | ---------- | ------------------------------ |
| 草稿    | 编辑、提交审核、删除 | 编制中                            |
| 待审核   | 查看、撤回      | 已提交审批，可撤回                      |
| 已生效   | 查看、变更      | 审批通过                           |
| 审核不通过 | 编辑、提交审核、删除 | 审批未通过，可编辑后重新提交或删除              |
| 已变更   | 编辑、提交审核、删除 | 已发起变更，可编辑后提交审核                 |
| 复核中   | 查看         | 采购服务中心审核中                      |
| 已复核   | 查看、变更      | 采购服务中心审核通过（系统层面等同于已生效）         |
| 已接收   | 查看         | 代理机构负责人将对应委托单指派给项目经理，项目经理接收委托单 |
| 已建项   | 查看         | 项目经理已关联到后续正式采购项目，仅可查看          |

**立项状态流转图**：

```
首次编制：
  草稿 → 提交审核 → 待审核 → 审核通过 → 已生效
                      ↘ 审核不通过 → 审核不通过 → 编辑 → 草稿
                      ↘ 撤回 → 草稿

变更流程：
  已生效/已复核 → 点击变更 → 已变更 → 提交审核 → 待审核 → 审核通过 → 已生效
                                              ↘ 审核不通过 → 审核不通过

复核流程（触发采购服务中心审核时）：
  待审核 → 本级/二级审批通过 → 复核中 → 采购服务中心审核通过 → 已复核
                                      ↘ 采购服务中心审核不通过 → 审核不通过

建项流程（非委托采购）：
  已生效/已复核 → 被后续采购项目关联使用 → 已建项

建项流程（委托采购）：
  已生效 → 代理机构负责人指派 → 已接收 → 被后续采购项目关联使用 → 已建项
```

## 五、页面设计

### 5.1 采购项目立项

**功能路径**：`采购系统 → 采购立项 → 采购项目立项`

**功能定位**：基于可执行计划中的子计划/项目计划，拆分到明细级别进行立项。

**核心规则**：
- 前置条件：必须关联可执行计划中的子计划/项目计划
- 关联层级：关联到明细级别，只要子计划/项目计划下还有未使用的明细，该计划就可选
- 明细选择：选择子计划/项目计划后，必须在其内部选择具体明细
- 项目类型一致性：同一立项中，所选明细的项目类型（工程/物资/服务）必须一致（提交时校验）
- 采购方式：拟采购方式自动带入（第一个子计划的拟采购方式），可编辑，以立项为准
- 立项后标记：明细被立项使用后，标记为"已使用"，不可再次用于立项
- 估算金额：取已选明细的估算金额之和

### 5.2 市场煤采购立项

**功能路径**：`采购系统 → 采购立项 → 市场煤采购立项`

**功能定位**：基于可执行电煤计划中的电煤计划，拆分到电煤明细级别进行立项。

**核心规则**：
- 前置条件：必须关联可执行电煤计划中的电煤计划（仅市场煤）
- 关联层级：关联到电煤明细级别
- 明细选择：选择电煤计划后，必须在其内部选择具体电煤明细
- 采购方式：拟采购方式自动带入，可编辑
- 立项后标记：电煤明细被立项使用后，标记为"已使用"，不可再次用于立项
- 变更规则：已生效的市场煤采购立项可变更，变更后需重新提交审核
- 撤回规则：待审核状态下可撤回，撤回后状态变为草稿
- 删除规则：草稿和审核不通过状态下可删除

#### 列表页

- **Tab 页签**：全部 / 草稿 / 待审核 / 已生效 / 已变更
- **搜索条件**：项目名称、采购方式、状态
- **列表字段**：序号、项目名称、项目编号、项目类型、组织形式、估算金额、采购方式、是否紧急采购、状态、创建时间
- **操作按钮**：新增市场煤采购立项

**操作按钮根据状态显示**：
- 草稿：编辑 / 删除
- 待审核：详情 / 撤回
- 已生效：查看 / 变更
- 已变更：编辑 / 提交审核
- 审核不通过：编辑 / 删除
- 已建项：查看

### 6.2 采购服务中心审核条件

当立项项目满足以下"应招未招"或"应公开未公开"条件时，在基础审批路径之后追加采购服务中心审核。

**注**：市场煤采购立项（电煤立项）不触发采购服务中心审核，仅走基础审批路径。

**招标限额标准**：

| 项目类型   | 招标限额            |
| ------ | --------------- |
| 工程     | ≥ 400万元         |
| 物资     | ≥ 200万元         |
| 服务     | ≥ 100万元         |

**触发条件判断规则**：

| 采购方式 | 关联字段值 | 达到招标限额（≥限额） | 未达到招标限额（<限额） |
|---------|-----------|-------------------|---------------------|
| 公开招标 | — | ❌ 不触发 | ❌ 不触发 |
| 邀请招标 | — | ✅ 触发 | ✅ 触发 |
| 询比采购 | 公开 | ✅ 触发 | ❌ 不触发 |
| 询比采购 | 邀请 | ✅ 触发 | ✅ 触发 |
| 谈判采购 | 公开 | ✅ 触发 | ❌ 不触发 |
| 谈判采购 | 邀请 | ✅ 触发 | ✅ 触发 |
| 竞价 | 公开 | ✅ 触发 | ❌ 不触发 |
| 竞价 | 邀请 | ✅ 触发 | ✅ 触发 |
| 直接采购 | — | ✅ 触发 | ❌ 不触发 |

**判断逻辑**：

1. 首先根据项目类型和估算金额判断是否达到招标限额
2. **达到限额时**：除公开招标外，其余采购方式均触发（询比/谈判/竞价的公开和邀请都触发，邀请招标触发，直接采购触发）
3. **未达到限额时**：仅邀请招标及"邀请"类方式（询比-邀请、谈判-邀请、竞价-邀请）触发
4. 直接采购仅在达到限额时触发，未达到限额不触发

**完整审批路径示例**：

```
应招未招/应公开未公开项目：
  待审核 → 本级审批通过 → 二级单位审批通过 → 复核中 → 采购服务中心审核通过 → 已复核
                                              ↘ 采购服务中心审核不通过 → 审核不通过

普通项目：
  待审核 → 本级审批通过 → 二级单位审批通过 → 已生效
```

**说明**：
- 预算金额取立项的估算金额
- 项目类型取立项的项目类型（工程/物资/服务）
- 关联字段值指采购方式对应的联动字段（询比方式/谈判方式/竞价方式）
