# 采购委托功能设计

> 设计日期：2026-06-08
> 最后修改：2026-06-17
> 状态：需求梳理完成

---

## 一、需求概述

### 1.1 功能定位

采购委托模块位于**采购立项**与**项目管理**之间，负责委托单的接收、分配、退回以及委托合同的管理，最终将符合条件的委托单交付到项目建项环节。

### 1.2 委托路径

| 路径   | 委托方               | 被委托方       | 流程起点                            |
| ---- | ----------------- | ---------- | ------------------------------- |
| 线上委托 | 采购单位创建采购立项，主动委托代理 | 代理单位（弹窗选择） | 采购立项审批通过 → 系统自动写入「指派委托单」未接收 Tab |
| 线下委托 | 采购单位线下委托          | 自建（当前用户单位） | 代理项目经理新建（新建委托单）→ 提交审核 → 审批通过自动进入「已接收」可建项 |

### 1.3 模块划分

```
采购委托管理
├── 1. 指派委托单    ← 指派员：接收 / 不接收 / 指派 / 撤回 / 退回   （仅线上）
├── 2. 新建委托单    ← 项目经理：新建(线下) / 撤回审核 / 删除 / 建项
├── 3. 接收委托单    ← 项目经理：查看 / 建项 / 退回                 （仅线上）
└── 4. 委托合同      ← 项目经理：创建 / 上传 / 建项时引用
```

审核入口：采购系统 → 任务中心

### 1.4 操作角色

| 角色 | 职责 | 系统角色 |
|------|------|---------|
| 代理机构负责人 / 项目指派员 | 接收委托单、指派、撤回、退回、不接收 | 待定（暂用 `AGENT_MANAGER`） |
| 代理项目经理 | 查看 / 建项 / 退回（接收委托单）、新建线下委托单（新建委托单）、管理委托合同 | `BUSINESS_USER` |

### 1.5 菜单结构

```
采购系统
├── 采购立项（已有）
├── 采购委托（本模块新增）
│   ├── 指派委托单     ← AGENT_MANAGER
│   ├── 新建委托单     ← BUSINESS_USER
│   ├── 接收委托单     ← BUSINESS_USER
│   └── 委托合同       ← BUSINESS_USER
└── 项目管理（已有，扩展建项入口）
    └── 我的项目 → 项目列表（新建弹窗新增「选择委托单」Tab）
```

### 1.6 审批流程

| 业务 | 审批路径 | 说明 |
|------|---------|------|
| 线下新建委托单 | 本级审批 | 审批通过后直接进入 PM「已接收」状态 |
| 委托合同 | 本级审批 | 审批通过后进入「已生效」状态 |

指派员接收/不接收/指派/退回/撤回，PM 接收/退回，均**不触发审批**，由操作人直接执行。

## 二、业务流程

### 2.1 线上委托流程

**采购立项（组织形式=委托采购）审批通过** → 系统自动写入「指派委托单」未接收 Tab。

**主链**：

```
未接收 ─[接收]→ 未指派 ─[指派]→ 已指派 ─[PM建项]→ 已建项
```

**委托单状态码（线上）**：

| 状态 | 状态码 | 说明 |
| --- | --- | --- |
| 未接收 | `UNCONFIRMED` | 采购立项审批通过，等待指派员接收/不接收 |
| 未指派 | `UNASSIGNED` | 指派员已接收，等待指派（含被 PM 退回后重新进入） |
| 已指派 | `ASSIGNED` | 已指派给 PM，等待建项/退回 |
| 已建项 | `PROJECT_CREATED` | 终态：已关联采购项目（线上+线下共用） |
| 已退回 | `REJECTED` | 终态：不接收/退回后移除（立项回草稿） |

**操作 → 状态映射**（线上委托唯一权威表，已含采购立项联动）：

| 操作 | 委托单状态 | 采购立项状态 |
| --- | --- | --- |
| 指派员接收 | 未接收 → 未指派 | 已生效 → 已接收 |
| 指派员不接收 | 未接收 → 已退回（移除） | 已生效 → 草稿 |
| 指派员指派 | 未指派 → 已指派 | 已接收 → 已指派 |
| 指派员退回 | 未指派 → 已退回（移除） | 已接收 → 草稿 |
| 指派员撤回指派 | 已指派 → 未指派 | 已指派 → 已接收 |
| PM退回 | 已指派 → 未指派 | 已指派 → 已接收 |
| PM建项 | 已指派 → 已建项 | 已指派 → 已建项 |

**关键规则**：
- **PM 直接对「已指派」做建项/退回**，无独立接收动作；撤回指派在 PM **建项前**可用。
- **不接收 / 指派员退回**后，采购立项回退到草稿、委托单已退回并从列表移除（不可继续操作）；采购单位需重新提报，系统生成新委托单。
- **撤回指派 / PM退回**后，委托单回到「未指派」可重新指派，立项回退到已接收。
- 采购立项状态链：**已生效 → 已接收 → 已指派 → 已建项**；线下建项无独立立项记录，不联动。

### 2.2 线下委托流程

```
项目经理新建委托单 → 填写立项信息 + 委托单信息 → 提交审核
    │ 本级审批
    ├── [审批通过]  →  直接进入 已接收 状态  →  新建委托单「已接收」Tab  →  可建项
    └── [审批不通过]  →  审核不通过 状态  →  新建委托单「审核不通过」Tab  →  可重新编辑
```

**委托单状态码（线下）**：

| 状态 | 状态码 | 说明 |
| --- | --- | --- |
| 草稿 | `DRAFT` | 编制中 |
| 待审核 | `PENDING_APPROVAL` | 审核中 |
| 审核不通过 | `APPROVAL_REJECTED` | 审核拒绝，可重新编辑 |
| 已接收 | `PM_RECEIVED` | 审批通过后进入，可建项 |
| 已建项 | `PROJECT_CREATED` | 终态：已关联采购项目（线上+线下共用） |

**线下委托特点**：
- **完全不进入指派委托单模块**，全程仅在「新建委托单」中流转
- 创建人即项目经理，无需接收；审批通过后状态直接为 **已接收**，自动出现在「新建委托单」已接收 Tab

### 2.3 委托合同流程

```
代理项目经理新建委托合同 → 填写合同信息 + 上传合同文件 → 提交审核
    │ 本级审批
    ├── [审批通过]  →  已生效  →  可在委托单 / 项目中关联引用
    └── [审批不通过]  →  审核不通过  →  可重新编辑
```

**关联引用**：委托合同在委托单主表「关联委托合同」字段引用，也可在项目建项时弹窗选择，均为手动，不自动绑定。

## 三、数据库表

### 3.1 委托单主表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 委托单名称 | 文本 | 是 | 线上：自动回显「计划名称 + 委托单」（可编辑）；线下：手动填写 |
| 委托单编号 | 文本 | 系统填写 | 系统自动生成，规则待定 |
| 委托来源 | 文本 | 系统填写 | 线上委托 / 线下委托，只读 |
| 关联采购立项 | 文本 | 系统填写 | 线上：关联的采购立项名称及编号（只读）；线下：创建时填写的项目名称及项目信息摘要 |
| 采购单位 | 文本/弹窗选择 | 是 | 线上：只读（采购立项创建单位）；线下：弹窗选择 |
| 代理单位 | 文本/弹窗选择 | 是 | 线上：弹窗选择；线下：只读（当前用户单位） |
| 联系人姓名 | 文本 | 是 | 默认：采购单位管理员姓名，可编辑 |
| 联系电话 | 文本 | 是 | 默认：采购单位联系电话，可编辑 |
| 关联委托合同 | 弹窗选择 | 否 | 从已生效委托合同中选择引用 |
| 备注 | 长文本 | 否 | |
| 采购委托单文件 | 文件 | 系统填写 | 线上：系统自动生成；线下：手动上传 |
| 附件 | 文件 | 否 | 支持多个，excel/pdf/word |
| 当前指派人 | 文本 | 冗余字段 | 最近一次指派的 PM 姓名 |
| 当前指派时间 | 日期时间 | 冗余字段 | 最近一次指派的时间 |
| 创建人/创建时间 | 系统填写 | | |
| 更新人/更新时间 | 系统填写 | | |

#### 3.1.1 线下委托额外字段（立项信息区域）

线下委托不经过采购立项环节，需在委托单内承载立项信息（字段定义引用《采购立项功能设计》3.1.1 节）：

| 字段分类   | 包含字段                                          |
| ------ | --------------------------------------------- |
| 项目基础信息 | 项目名称、项目编号、项目类型、估算金额                           |
| 实施信息   | 项目所属区域、项目实施地点、项目性质                            |
| 采购配置   | 采购方式、资审方式、邀请供应商、询比方式、谈判方式、直接采购属性、竞价方式（及其联动规则） |
| 分类与要求  | 主要标的类型、主要标的类别占比、本项目行业分类                       |
| 文件与描述  | 项目审批/核准/备案文件、项目概况                             |
| 供应商要求  | 供应商基本要求、资质要求、业绩要求、其他要求                        |
| 其他     | 是否协同项目、协同目录、立项备注、附件                           |

**说明**：
- 不含"组织形式"（线下固定为委托采购）
- 不含"立项联系人/电话/提报单位"（已由委托单主表字段覆盖）
- 不含委托单自身字段（避免自引用）
