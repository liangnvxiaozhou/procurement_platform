## 核心设计决策

### 1. 架构简化
- **无独立规则库页面**：去掉"系统配置 → 收费规则"的独立页面
- **规则归属租户**：收费规则以租户ID为外键，每个租户拥有自己的招标+非招规则
- **唯一入口**：租户列表操作列增加"平台使用费"按钮，打开配置抽屉完成全部操作

### 2. 数据结构

#### 2.1 平台使用费规则主表（platform_fee_rule）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 规则名称 | 文本 | ✅ | 用户手填 |
| 规则编号 | 文本 | ✅系统自动 | 前缀+序号，招标类 `FEE-ZB-XXX`，非招类 `FEE-FZB-XXX` |
| 计算方式 | 文本 | ✅系统自动 | 固定为"梯度进制"，只读 |
| 计算金额 | 下拉选择 | ✅ | 计费依据：预算价 / 成本价 |
| 项目类别 | 下拉选择 | ✅ | 招标 / 非招 |
| 租户ID | 外键 | ✅系统自动 | 所属租户，打开抽屉时自动关联 |
| 创建人 | 文本 | ✅系统自动 | 创建用户姓名 |
| 创建时间 | 日期时间 | ✅系统自动 | yyyy-mm-dd HH:mm:ss |
| 修改人 | 文本 | ✅系统自动 | 最后更新用户姓名 |
| 修改时间 | 日期时间 | ✅系统自动 | yyyy-mm-dd HH:mm:ss |

#### 2.2 阶梯规则表（platform_fee_tier）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 规则ID | 外键 | ✅ | 关联主表 |
| 序号 | 整数 | ✅系统自动 | 按最小值升序排序 |
| 最小值（万元） | 数字 | ✅ | 左闭，含本值 |
| 最大值（万元） | 数字 / 特殊值 | ✅ | 右开，不含本值；最后一档可选"无上限" |
| 收费金额（元） | 数字 | ✅ | 固定金额，精确到分 |

### 3. 页面设计

**入口**：租户列表操作列，增加"平台使用费"按钮

**配置抽屉结构**：
- 抽屉标题：平台使用费配置
- 分两个区域：招标类收费规则、非招标类收费规则
- 每个区域包含：基本信息表单 + 阶梯规则表格
- 阶梯规则支持：添加档位（弹窗）、编辑档位（弹窗）、删除档位
- 底部：取消 / 保存

**阶梯弹窗字段**：
- 最小值（万元）：第一档固定0只读，后续自动带入上一档最大值
- 最大值（万元）：手动填写，或勾选"无上限"
- 收费金额（元）：手动填写

### 4. 业务规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R1 | 最小值起始 | 第一个档位的最小值固定为 0，只读 |
| R2 | 阶梯连续性 | 后续档位的最小值自动带入上一档的最大值，只读 |
| R3 | 区间有效 | 最大值必须大于最小值 |
| R4 | 无上限唯一 | "无上限"只能出现在最后一个档位，且整个规则只能有一个 |
| R5 | 金额正数 | 收费金额必须为正数，精确到分 |
| R6 | 至少一档 | 每套规则必须至少包含一个阶梯档位 |
| R7 | 收费递增提示 | 当收费金额未随区间增大而递增时，弹出确认提示（不强制） |
| R8 | 编号唯一 | 规则编号全局唯一 |
| R9 | 租户必填 | 每个租户必须配置招标类和非招标类两套规则（可后续补配） |
| R10 | 切换即时生效 | 租户修改收费规则后，新立项项目按新规则计费；已立项项目不受影响（已快照） |

### 5. 与其他功能的关系

**项目立项**：
- 标段包中的"平台使用费收费规则"字段，自动读取当前租户关联的收费规则
- 根据采购方式判断显示招标类或非招标类规则
- 字段为只读

**全局日志审计**：
- 规则的新建、编辑操作记录在控制中心的全局操作日志中
